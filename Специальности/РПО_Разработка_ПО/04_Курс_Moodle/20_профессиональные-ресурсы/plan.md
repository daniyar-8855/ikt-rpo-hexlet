**Дисциплина:** Применение информационно-коммуникационных и цифровых технологий  
**Тема занятия:** Профессиональные ресурсы разработчика (Stack Overflow, MDN, GitHub)

**Дата:** «___» __________ 20__ г.

## 1. Общие сведения

- **Курс, группа:** ____________________ (квалификация 4S06130103 «Разработчик ПО»)
- **Тип занятия:** комбинированное
- **Продолжительность:** 2 часа (90 мин)
- **Результат обучения:** РО 2.2 · **Связь с ПМ:** Разработка программных модулей

## 2. Цели, задачи

**Цель:** Сформировать умение выбирать профессиональные ресурсы разработчика под задачу, грамотно искать и оценивать найденное, ориентироваться в GitHub.

**Задачи:**
- показать назначение документации (MDN, docs.python.org), Stack Overflow и GitHub;
- научить оценивать ответ на Stack Overflow по дате, голосам и версии;
- сформировать навык постановки хорошего вопроса на Stack Overflow;
- раскрыть роль GitHub как хранилища кода и портфолио разработчика.

## 3. Ожидаемые результаты

Обучающийся:
- выбирает правильный ресурс под конкретную задачу;
- оценивает достоверность ответа на Stack Overflow;
- называет элементы хорошего вопроса на Stack Overflow;
- объясняет назначение GitHub (версии, история, совместная работа, портфолио).

## 4. Необходимые ресурсы

- Курс в Moodle: страница «Теория», тест и задание занятия 20.
- Схемы: «Какой ресурс для чего», «Как задать хороший вопрос на Stack Overflow» (папка assets).
- Компьютер, проектор, доступ к Moodle и интернету (MDN, Stack Overflow, GitHub).

## 5. Ход занятия

| Этап / Время | Действия педагога | Действия обучающегося |
|---|---|---|
| Организационный момент (5 мин) | Приветствие, проверка готовности и присутствующих | Занимают места, готовятся к занятию |
| Мотивация (10 мин) | Вводит практической ситуацией («код упал, ошибка непонятна»), задаёт вопросы | Обсуждают, предлагают, где искать ответ |
| Целеполагание (5 мин) | Объявляет тему и цели занятия | Записывают тему, формулируют ожидания |
| Изучение нового материала (35 мин) | Раскрывает три опоры (документация, Stack Overflow, GitHub), демонстрирует схемы, показывает поиск и оценку ответа, приводит мини-кейс | Слушают, работают со схемами, задают вопросы |
| Закрепление (15 мин) | Организует практику (выбор ресурса, элементы хорошего вопроса) и разбор типичной ошибки | Выполняют задания, отвечают |
| Контроль (10 мин) | Запускает тест в Moodle | Проходят тест самопроверки |
| Итог и рефлексия (5 мин) | Возвращает к вопросам: что делать, где применяется, как не ошибиться | Отвечают, подводят итог |
| Задание (5 мин) | Поясняет вариантное задание и критерии | Записывают, уточняют |
