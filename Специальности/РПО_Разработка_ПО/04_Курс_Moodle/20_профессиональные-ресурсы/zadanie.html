Задание к занятию 20. Решение задачи через ресурсы разработчика

Вариант — по номеру в журнале (своя задача на ~5 человек). № 1–5 → В1 · 6–10 → В2 · 11–15 → В3 · 16–20 → В4 · 21–25 → В5 · 26–30 → В6.

Обязательная часть (до 85 баллов)

  1. Найди решение задачи своего варианта, используя документацию и Stack Overflow.
  2. Приведи ссылку на документацию и на ответ SO (с датой и рейтингом).
  3. Объясни решение своими словами (покажи, что понял код, а не просто скопировал).
  4. Опиши, как бы ты сохранил это решение в репозитории на GitHub (структура, README).
Продвинутая часть (+15 баллов): создай учебный репозиторий на GitHub с этим решением и README, приложи ссылку. Защити устно.

Задачи по вариантам

  • В1. Прочитать файл и посчитать строки (Python).
  • В2. Сделать кнопку с обработчиком клика (JS).
  • В3. Отсортировать список словарей по ключу (Python).
  • В4. Центрировать блок по центру страницы (CSS).
  • В5. Отправить GET-запрос и вывести ответ (JS fetch или Python requests).
  • В6. Создать ветку, внести изменение, слить в main (git).

СРС (самостоятельная работа, 2 ч)

Тема СРС: Решение задачи с помощью Stack Overflow/MDN (отчёт).
Задание: Реши учебную задачу, опираясь на Stack Overflow и MDN, оформи отчёт с источниками. Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: указаны источники, описано решение, есть собственный вывод.
