Библиотеки Pandas и NumPy

Считать данные «вручную» циклами хорошо для понимания, но в реальности данных тысячи строк. Тогда подключают готовые библиотеки. NumPy быстро считает над массивами чисел, Pandas работает с таблицами (как Excel, но в коде). Одна строка заменяет целый цикл.

Чему ты научишься: зачем нужны NumPy и Pandas; что такое DataFrame; считать показатели по столбцу и фильтровать строки одной командой.

NumPy: массивы чисел

Массив похож на список, но операции считаются быстро и сразу над всеми элементами:

import numpy as np
ceny = np.array([1500, 2000, 1800])
print(ceny.mean())   # 1766.6 — среднее
print(ceny.sum())    # 5300
print(ceny * 2)      # [3000 4000 3600]

Не нужен цикл — mean(), sum(), умножение применяются ко всему массиву.

Pandas: таблицы данных

Главный объект — DataFrame (таблица со столбцами и строками):

import pandas as pd
data = {"tovar": ["Мышь", "Клав.", "Монитор"],
        "cena":  [1500, 2000, 9000]}
df = pd.DataFrame(data)
print(df["cena"].mean())   # среднее по столбцу
print(df["cena"].max())    # максимум
Метод столбцаЧто делает
.mean()среднее
.sum()сумма
.max() / .min()наибольшее / наименьшее
.count()количество значений

Фильтрация строк

dorogie = df[df["cena"] >= 2000]
print(dorogie)

Одна строка вместо цикла с if и append.

Мини-кейс

В таблице 1000 заказов со столбцом summa. Средний чек — df["summa"].mean(); число заказов дороже 5000 — len(df[df["summa"] > 5000]). Две строки вместо длинного цикла.

Ошибка: забыть import pandas / numpy.
Почему: без импорта Python не знает этих имён (NameError).
Как правильно: импорт в начале файла.
Ошибка: опечатка в имени столбца — df["Cena"] вместо df["cena"].
Почему: имена чувствительны к регистру (KeyError).
Как правильно: проверить имена через df.columns.
Итог — что теперь делать:
  • NumPy — для быстрых расчётов над массивами чисел;
  • Pandas — для таблиц: DataFrame = таблица со столбцами;
  • считай показатели по столбцу: mean(), sum(), max();
  • фильтруй строки: df[df["поле"] > значение].
