1. Введение: зачем тебе эта тема

Расскажу, как это выглядит на самом деле — не в теории, а в обычный рабочий вторник. Тебе в мессенджер падает файл orders.csv: тысяча строк заказов интернет-магазина — товар, количество, сумма, город, дата. И задача в одну строчку: «к завтрашнему утру три цифры — средний чек, сколько заказов дороже 5000 и какие товары берут чаще всего». Звучит как работа на вечер, и внутри уже начинает ныть.

Дальше происходит развилка, и я видел её десятки раз. Один студент открывает пустой файл и первым делом тянется к циклу: пройтись for-ом по каждой строке, складывать суммы в переменную, потом делить на количество, завести счётчик для дорогих, собирать списки через append. Это работает — ровно на тех десяти строках, что он проверил глазами. На тысяче такой код становится длинным, медленным и хрупким: одна перепутанная переменная или сбитый отступ — и отчёт молча выдаёт неверное число. Самое противное, что программа при этом не падает. Она не кричит «ошибка», она просто спокойно считает не то, а ты сдаёшь начальнику красивые, но неправильные цифры и узнаёшь об этом через неделю от бухгалтерии.

Второй студент пишет три строки: загрузил файл, взял столбец summa, посчитал среднее — и через минуту отправляет ответ. Он не умнее первого. Он просто знает, что в реальной работе циклы для таких задач не пишут. Подключают готовые библиотеки: NumPy — для быстрых расчётов над числами, и Pandas — для работы с таблицами. С ними одна строка заменяет целый цикл: df["summa"].mean() — и средний чек готов; df[df["summa"] > 5000] — и все дорогие заказы отобраны. Отчёт, на который заложен вечер, собирается за пару минут, и его не стыдно перепроверить. Посмотри на схему «DataFrame — это таблица в коде» (приложение А): именно так Pandas видит твой файл — как таблицу со столбцами и строками, к каждому столбцу которой можно обратиться по имени.

Эта тема — про то, чтобы гарантированно оказаться вторым студентом. И почему это важно лично тебе, а не «аналитику где-то рядом». Разработчик ПО постоянно достаёт данные из базы или файла и считает по ним показатели: сколько пользователей активны, какая средняя длительность сессии, какие товары не продаются, где проседает конверсия. Это не отдельная профессия — это часть твоей повседневной работы с любым проектом, где есть данные (а он такой почти всегда). В вакансиях на Python-разработчика, и тем более на дата-инженера или ML-инженера, Pandas и NumPy стоят в графе «обязательно», а не «будет плюсом» [4]. И это же — фундамент для следующих тем курса: визуализации данных (занятие 31) и анализа данных с помощью ИИ (занятие 32). Освоив Pandas сейчас, ты получаешь инструмент, которым будешь пользоваться всю профессиональную жизнь.

3. Ключевые понятия

  • Библиотека — готовый набор кода, который ты подключаешь к программе, чтобы не писать всё с нуля [5].
  • NumPy — библиотека для быстрых вычислений над массивами чисел [6].
  • Массив (ndarray) — основной объект NumPy: упорядоченный набор чисел одного типа, над которым операции выполняются сразу над всеми элементами.
  • Векторная операция — действие, применяемое ко всему массиву целиком, без явного цикла.
  • Pandas — библиотека для работы с табличными данными [5].
  • DataFrame — главный объект Pandas: таблица с именованными столбцами и строками.
  • Series — один столбец DataFrame; набор значений с подписями (индексом).
  • Фильтрация (boolean indexing) — отбор строк таблицы по логическому условию.
  • Маска — столбец из True/False, по которому фильтруются строки.
  • Псевдоним (alias) — короткое имя при импорте: import numpy as np, import pandas as pd.

4. Подробная теория

4.1. Что такое библиотека и почему «не пиши то, что уже написано»

Прежде чем разбирать NumPy и Pandas по отдельности, разберись с самим словом «библиотека» — иначе строка import в начале файла так и останется для тебя магическим заклинанием, которое «надо написать, а зачем — непонятно».

Библиотека — это чужой готовый код, который ты подключаешь к своему. Кто-то однажды написал, отладил и годами проверял функции для работы с числами или таблицами, упаковал их и выложил в общий доступ. Ты пишешь одну строку import — и эти функции становятся доступны в твоей программе, как будто ты написал их сам. Тебе не нужно заново изобретать, как быстро посчитать среднее по миллиону чисел так, чтобы это не тормозило: это уже сделали за тебя, причём заведомо лучше, чем сможет новичок, — код Pandas и NumPy вычитан и оттестирован тысячами людей.

Аналогия, которая всё ставит на место. Когда тебе нужно забить гвоздь, ты не выплавляешь молоток из руды — берёшь готовый из ящика. Библиотека — это готовый инструмент в твоём ящике. Python из коробки умеет немного: списки, циклы, базовая математика. А специализированные задачи — расчёты, таблицы, графики, веб — закрываются библиотеками. И профессионализм разработчика во многом состоит именно в том, чтобы знать, какая библиотека для какой задачи существует, и не переписывать руками то, что уже написано и вылизано до тебя.

Подключают библиотеку командой import. Почти всегда — с коротким псевдонимом, чтобы не печатать длинное имя в каждой строке:

import numpy as np      # теперь NumPy доступен как np
import pandas as pd     # теперь Pandas доступен как pd

Псевдонимы np и pd — это не случайность и не твой личный вкус. Это общепринятое соглашение всего сообщества [5; 6]. В любом учебнике, в любом ответе на Stack Overflow, в любом чужом проекте ты увидишь именно np и pd. Пиши так же — тогда твой код будет понятен другим, а чужой код будет понятен тебе с первого взгляда. Эти две строки почти всегда стоят в самом начале файла. Забыть их — самая частая ошибка новичка, к которой мы вернёмся в разделе 6, и там же увидим, как именно на неё ругается Python.

4.2. NumPy: массив чисел и операции над всеми элементами сразу

NumPy (от Numerical Python) — библиотека для быстрых вычислений над числами [6]. Её главный объект — массив, по-английски ndarray (n-dimensional array, многомерный массив). Создаётся он из обычного списка функцией np.array(). Давай сразу посмотрим, как это выглядит на экране, чтобы не было абстракции:

import numpy as np
ceny = np.array([1500, 2000, 1800])
print(ceny)        # [1500 2000 1800]
print(type(ceny))  # <class 'numpy.ndarray'>

Прочитаем вывод построчно — это навык, который мы будем прокачивать всю тему. Первый print показал [1500 2000 1800] — обрати внимание: числа разделены пробелами, а не запятыми, как в обычном списке Python. Это первая внешняя примета, что перед тобой не список, а массив NumPy. Второй print прямым текстом подтверждает тип: numpy.ndarray. На первый взгляд массив похож на список — но разница не в том, как он выглядит, а в том, что с ним можно делать.

Главная суперсила NumPy в том, что операция применяется сразу ко всему массиву, без цикла. Это называется векторной операцией. Сравни два способа удвоить каждую цену. Сначала так, как делал бы новичок без NumPy:

ceny = [1500, 2000, 1800]
udvoennye = []
for cena in ceny:           # перебираем по одному
    udvoennye.append(cena * 2)
print(udvoennye)            # [3000, 4000, 3600]

Четыре строки, явный цикл, вспомогательный пустой список, append на каждом шаге. Всё это места, где можно опечататься. А теперь то же самое с NumPy:

ceny = np.array([1500, 2000, 1800])
print(ceny * 2)             # [3000 4000 3600]

Одна строка. Никакого цикла, никакого вспомогательного списка. Ты просто пишешь ceny * 2, и NumPy сам применяет умножение к каждому элементу. Вот что здесь важно понять до конца: с обычным списком Python [1500, 2000, 1800] * 2 дало бы совсем другое — список удвоился бы по длине ([1500, 2000, 1800, 1500, 2000, 1800]), а не по значению. NumPy же понимает * 2 как «умножь каждое число». Это и есть векторная операция: пишешь действие над всем массивом, а NumPy раскрывает его до действий над каждым элементом. Посмотри на схему «NumPy: операция применяется ко всем элементам сразу» (приложение Б): сверху исходные числа, в центре — операция ceny * 2, снизу результат, где каждый элемент уже изменён. Стрелки показывают, что одна команда «спускается» сразу ко всем ячейкам.

Так же работают и расчёты целиком по массиву — для них у NumPy есть готовые методы:

ceny = np.array([1500, 2000, 1800])
print(ceny.mean())   # 1766.67 — среднее
print(ceny.sum())    # 5300       — сумма
print(ceny.max())    # 2000       — максимум
print(ceny.min())    # 1500       — минимум

Разберём по шагам, что происходит в строке ceny.mean(), — это типовой шаблон «объект → точка → метод», который ты будешь писать постоянно:

  1. ceny — это массив [1500, 2000, 1800].
  2. Точка означает «вызови у этого объекта его метод».
  3. mean() — метод «посчитай среднее арифметическое». Скобки означают вызов; забудешь скобки — получишь не число, а сам метод как объект.
  4. NumPy сам складывает все элементы (1500 + 2000 + 1800 = 5300) и делит на их количество (5300 / 3 = 1766.67).
  5. Результат возвращается, и print его выводит.

Тебе не нужно писать ни суммирование, ни деление, ни подсчёт количества — всё это уже внутри метода mean(). Одна команда заменяет несколько строк ручного кода. И заметь честную деталь: среднее вышло 1766.67 — число не круглое, с бесконечным «хвостом» (5300 не делится на 3 нацело). NumPy не округляет за тебя молча — при выводе ты видишь дробь; в реальном отчёте округление до копеек (двух знаков) ты делаешь уже сам, осознанно, а не полагаешься на «как получилось».

Почему это не просто короче, а ещё и быстрее. Здесь важный момент, который отличает понимание от зубрёжки. Когда ты пишешь цикл for на чистом Python, каждый его шаг — это работа интерпретатора Python: он по очереди берёт элемент, выполняет действие, берёт следующий. Это гибко, но медленно. NumPy же хранит числа компактно, одним куском памяти, и выполняет операции над ними «оптом» на низком уровне — внутри библиотека написана на быстром языке Си [6]. Поэтому на массиве из миллиона чисел arr.sum() отработает в десятки раз быстрее, чем ручной цикл. На трёх числах разницы ты не заметишь глазом — но в реальной работе данные большие, и тогда векторные операции NumPy превращаются из «просто красивее» в «единственно разумно».

Запомни вывод этого раздела: NumPy — это про числа и скорость. Один массив, одна команда — расчёт над всеми элементами сразу.

4.3. Pandas: таблица как объект DataFrame

NumPy отлично работает с «голыми» числами. Но реальные данные почти всегда живут в таблицах: строки — это записи (заказ, клиент, товар), столбцы — это признаки (сумма, имя, дата, город). Для таблиц есть Pandas — библиотека, построенная поверх NumPy и специально заточенная под табличные данные [5].

Главный объект Pandas — DataFrame. Проще всего понять его так:

DataFrame — это «Excel внутри кода»: таблица с именованными столбцами и пронумерованными строками, к каждому столбцу которой можно обратиться по имени и сразу посчитать по нему показатели.

Создать DataFrame можно из обычного словаря, где ключ — это имя столбца, а значение — список ячеек этого столбца:

import pandas as pd
data = {
    "tovar": ["Мышь", "Клав.", "Монитор"],
    "cena":  [1500,   2000,    9000],
}
df = pd.DataFrame(data)
print(df)

Вывод будет выглядеть как настоящая таблица:

     tovar  cena
0     Мышь  1500
1    Клав.  2000
2  Монитор  9000

Разбери эту картинку внимательно, она ключевая — тут не «текст с числами», а структура:

  • Столбцы — tovar и cena. Их имена ты задал ключами словаря. На схеме (приложение А) это синие шапки сверху.
  • Строки — три записи о товарах. На схеме это голубые ячейки под шапкой.
  • Индекс — числа 0, 1, 2 слева. Pandas сам пронумеровал строки, ты их не писал. По этому номеру можно обратиться к конкретной строке. Обрати внимание: нумерация с нуля, как везде в Python, а не с единицы, — новичок на этом иногда спотыкается.

То есть DataFrame — это не «куча данных», а структура с понятными координатами: «столбец cena, строка 2» — это цена монитора, 9000. Ровно как в Excel, где ячейка адресуется столбцом и строкой, только адресуешься ты не мышкой, а именем и индексом.

4.4. Доступ к столбцу: df["имя"] и что это возвращает

Главная операция в Pandas, которую ты будешь делать буквально в каждом скрипте, — обращение к столбцу по имени. Делается это так:

print(df["cena"])

Вывод:

0    1500
1    2000
2    9000
Name: cena, dtype: int64

Прочитаем ответ, а не просто пробежим глазами. Запись df["cena"] означает: «возьми из таблицы df весь столбец с именем cena». В ответ Pandas возвращает один столбец — объект, который называется Series. Series — это, по сути, столбец как список чисел, но «умный»: у него есть подписи строк (тот самый индекс 0, 1, 2 слева) и встроенные методы расчёта. Последняя строка вывода тоже не мусор, её полезно читать: Name: cena — имя столбца, dtype: int64 — тип данных, целые числа. Если бы вместо чисел там оказался текст, ты бы увидел dtype: object — и это первый сигнал, что по такому столбцу mean() не посчитается. На схеме (приложение А) зелёный блок внизу как раз показывает: df["cena"] → весь столбец как список чисел.

И вот здесь Pandas и NumPy соединяются в одно. Раз столбец df["cena"] ведёт себя как массив чисел, к нему применимы те же методы расчёта — сразу по всему столбцу, без цикла:

print(df["cena"].mean())   # 4166.67 — средняя цена
print(df["cena"].sum())    # 12500      — сумма всех цен
print(df["cena"].max())    # 9000       — самый дорогой
print(df["cena"].min())    # 1500       — самый дешёвый
print(df["cena"].count())  # 3          — сколько значений в столбце

Здесь та же честная деталь про округление, что и в разделе 4.2: (1500 + 2000 + 9000) / 3 = 12500 / 3 = 4166.666…, бесконечная дробь. В выводе ты увидишь длинный «хвост» (4166.666666...); я записал его как 4166.67, округлив до двух знаков сам, — но помни, что настоящий print покажет полную дробь, а решение, до скольких знаков округлить, всегда за тобой.

Разбери цепочку df["cena"].mean() по частям — это типовой шаблон, который ты будешь писать чаще всего:

  1. df — твоя таблица.
  2. ["cena"] — берём из неё столбец cena (получаем Series).
  3. .mean() — у этого столбца вызываем метод «посчитай среднее».
  4. Результат — одно число, среднее по столбцу.

Запомни эти пять методов столбца, они закрывают большинство повседневных задач:

МетодЧто считаетПример из жизни
.mean()среднее арифметическоесредний чек
.sum()сумму всех значенийобщая выручка
.max()наибольшее значениесамый дорогой заказ
.min()наименьшее значениесамый дешёвый заказ
.count()количество значенийсколько всего заказов

Отдельно про разницу между .sum() и .count(), её путают чаще всего: .sum() складывает сами числа (например, сумму всех цен), а .count() считает, сколько значений в столбце (сколько строк). Для столбца цен [1500, 2000, 9000] сумма равна 12500, а количество — 3. Спутаешь их в отчёте — и вместо «продано 3 товара» напишешь «продано 12500 товаров». Проверяй себя вопросом: мне нужна денежная сумма или число записей?

4.5. Фильтрация строк: оставить только то, что нужно

До сих пор мы считали по всему столбцу. Но в работе чаще нужно сначала отобрать строки по условию, а потом считать. «Оставь только дорогие заказы», «покажи товары дороже 5000», «возьми заказы из Алматы». В Pandas это делается одной командой — без цикла и без if.

Сначала посмотри, как это выглядело бы без Pandas, руками:

dorogie = []
for i in range(len(df)):           # перебираем строки по индексу
    if df["cena"][i] >= 2000:      # проверяем условие
        dorogie.append(df.iloc[i]) # добавляем подходящие

Длинно, и в каждой строке — шанс ошибиться: перепутать индекс, забыть append, промахнуться со знаком сравнения. А теперь то же самое в Pandas:

dorogie = df[df["cena"] >= 2000]
print(dorogie)

Вывод:

     tovar  cena
1    Клав.  2000
2  Монитор  9000

Осталось две строки из трёх — те, где цена ≥ 2000. «Мышь» за 1500 отфильтрована. Признаю честно: запись df[df["cena"] >= 2000] поначалу выглядит странно — таблица внутри квадратных скобок самой таблицы, глаз спотыкается. Поэтому разберём её не целиком, а в два шага, как её и стоит читать в голове.

Выполни внутреннюю часть отдельно — это половина всего фокуса:

print(df["cena"] >= 2000)

Вывод:

0    False
1     True
2     True
Name: cena, dtype: bool

Видишь, что произошло? Сравнение df["cena"] >= 2000 — это тоже векторная операция, ровно как ceny * 2 из раздела 4.2: оно применилось к каждой ячейке столбца и вернуло не одно True/False, а целый столбец из True/False. Заметь dtype: bool в конце — тип теперь логический, а не числовой. Это называется маской: для каждой строки записано, проходит она условие (True) или нет (False).

Теперь второй шаг. Когда ты пишешь df[маска], Pandas оставляет только те строки, против которых стоит True, и выбрасывает строки с False. Собираем оба шага вместе:

  1. df["cena"] >= 2000 строит маску [False, True, True];
  2. df[...] применяет маску и оставляет строки 1 и 2 (те, где True).

Это называется булева индексация (boolean indexing) [5]. Одна короткая запись делает то, на что в ручном коде ушёл бы цикл с условием. Меньше кода — меньше места для ошибки. И как только ты научился читать эту запись в два шага («сначала маска, потом отбор»), она перестаёт пугать и становится твоим главным рабочим инструментом.

4.6. Складываем всё вместе: мини-кейс с заказами

Вернёмся к задаче из введения — той самой, что упала в мессенджер: таблица заказов со столбцом summa, нужны средний чек и число заказов дороже 5000. Теперь у тебя есть все детали, чтобы решить её красиво:

import pandas as pd

df = pd.read_csv("orders.csv")           # 1. загрузили таблицу из файла
sredniy = df["summa"].mean()             # 2. средний чек — одной командой
mnogo = len(df[df["summa"] > 5000])      # 3. число дорогих заказов

print("Средний чек:", sredniy)
print("Дорогих заказов:", mnogo)

Разбор по шагам — что именно произошло:

  1. pd.read_csv("orders.csv") — Pandas сам читает файл CSV, разбирает его на столбцы и строки и превращает в DataFrame. Никаких ручных циклов чтения строк, никакого разбора запятых руками — всё внутри одной функции.
  2. df["summa"].mean() — берём столбец summa и считаем по нему среднее. Это весь твой «средний чек».
  3. df[df["summa"] > 5000] — фильтруем строки, где сумма больше 5000 (та самая маска из 4.5); len(...) считает, сколько таких строк осталось.

Три строки осмысленного кода вместо вечера с циклами. И вот что тут по-настоящему ценно: это масштабируется бесплатно. Что на тысяче заказов, что на миллионе — код ровно тот же, меняется только размер файла, а не твоя программа. С ручным циклом ты бы на миллионе строк ещё и ждал результата заметное время; Pandas отдаст его почти мгновенно.

4.7. Когда NumPy, а когда Pandas

Раз есть две библиотеки, у тебя закономерно возникнет вопрос: какую брать? Правило простое.

  • NumPy — когда у тебя однородные числа и нужны быстрые расчёты: один длинный ряд значений, матрица, вычисления для математики и машинного обучения. NumPy не знает про «имена столбцов» — он про чистые числа и скорость.
  • Pandas — когда у тебя таблица с разными столбцами: имена, суммы, даты, города вперемешку. Pandas даёт имена столбцам, удобный доступ по имени, чтение из файлов, фильтрацию.

На практике они работают в паре, а не соперничают: Pandas хранит данные таблицы, а внутри для расчётов опирается на NumPy [5; 6]. Поэтому методы .mean(), .sum() и векторные операции выглядят одинаково и там, и там — это не совпадение, а одна и та же машина расчётов под капотом. Для тебя как начинающего ориентир такой: данные пришли таблицей (файл, выгрузка из базы) — бери Pandas; нужно посчитать по чистому ряду чисел или ты идёшь в сторону ML — бери NumPy. А со временем ты перестанешь про это думать и будешь переключаться между ними, не задумываясь.

5. Разобранные примеры

Пример 1 — средний балл группы (NumPy). Дан список оценок за контрольную. Нужно среднее и максимум.

import numpy as np
ocenki = np.array([4, 5, 3, 5, 4, 2, 5])
print("Средний балл:", ocenki.mean())   # 4.0
print("Максимум:", ocenki.max())        # 5
print("Двоек:", (ocenki == 2).sum())    # 1

Разбор. Первые две команды — обычные расчёты по массиву, тут всё прямо. Третья строка хитрее и показывает мощь векторных операций во всей красе: ocenki == 2 строит маску [False, False, False, False, False, True, False], а .sum() складывает её. Ключевой трюк: в Python True считается за 1, а False за 0, поэтому сумма маски — это ровно количество двоек. Так одной строкой, без всякого счётчика и цикла, считают, сколько элементов удовлетворяют условию. Приём стоит запомнить намертво — «маска плюс .sum()» будет попадаться постоянно.

Пример 2 — отчёт по заказам (Pandas).

import pandas as pd
data = {
    "tovar": ["Мышь", "Клав.", "Монитор", "Ноутбук"],
    "cena":  [1500,    2000,    9000,      45000],
    "kol":   [10,      5,        3,         1],
}
df = pd.DataFrame(data)

print("Средняя цена:", df["cena"].mean())          # 14375.0
print("Всего товаров на сумму:", (df["cena"] * df["kol"]).sum())
dorogie = df[df["cena"] >= 9000]
print(dorogie[["tovar", "cena"]])

Разбор. Здесь сразу несколько приёмов в одном месте. df["cena"] * df["kol"] — поэлементное умножение двух столбцов (цена × количество для каждой строки по отдельности), а .sum() затем суммирует результат — получаем стоимость всех позиций на складе. Фильтр df["cena"] >= 9000 оставляет монитор и ноутбук. И новая деталь: запись dorogie[["tovar", "cena"]] — обрати внимание на двойные квадратные скобки. Одинарные скобки df["cena"] берут один столбец (Series), а двойные df[["tovar", "cena"]] — несколько столбцов сразу (снова DataFrame). Так строят аккуратный отчёт только из нужных колонок, отбросив лишнее. Спутаешь одинарные и двойные — получишь либо ошибку, либо не тот объект, так что это различие держи в голове.

Пример 3 — проверка имён столбцов перед работой.

import pandas as pd
df = pd.read_csv("orders.csv")
print(df.columns)   # Index(['tovar', 'summa', 'gorod'], dtype='object')
print(df.head())    # первые 5 строк — посмотреть, как выглядят данные

Разбор. Прежде чем считать хоть что-то, всегда смотри, что за данные пришли, — это привычка, которая экономит часы. df.columns показывает точные имена столбцов; копируй имя оттуда, а не пиши по памяти — это спасает от ошибки регистра из раздела 6 (в реальном файле столбец может называться Summa, SUMMA или summa , с хвостовым пробелом, и ты об этом узнаешь только отсюда). df.head() показывает первые пять строк — так ты сразу видишь, в каком виде данные и нет ли в них мусора: пустых ячеек, текста вместо чисел, странных значений. Эти две команды — обязательный первый шаг при работе с любым новым файлом, и опытный человек делает их на автомате, ещё до того как начнёт что-либо считать.

6. Частые ошибки и как их избежать

  • Ошибка: забыл import numpy as np или import pandas as pd. Почему: без импорта Python не знает имён np и pd и падает с NameError: name 'pd' is not defined. Это не «код сломался» — это прямая подсказка: имя pd не определено, потому что ты его не подключил. Как правильно: ставь обе строки импорта в самое начало файла, до любого использования библиотек.
  • Ошибка: обращение к столбцу с другим регистром — df["Cena"], когда столбец называется cena. Почему: имена столбцов в Pandas чувствительны к регистру; "Cena" и "cena" — для Pandas разные имена, и он честно отвечает KeyError: 'Cena' (нет такого ключа). Как правильно: проверяй точные имена через df.columns и копируй имя оттуда, а не набирай по памяти.
  • Ошибка: путать .sum() и .count(). Почему: .sum() складывает сами значения, а .count() считает их количество — это разные числа, и подмена одного другим тихо портит отчёт. Как правильно: спроси себя — «на какую сумму?» это .sum(); «сколько всего?» это .count().
  • Ошибка: писать цикл for там, где хватит одной векторной операции. Почему: код становится длиннее, медленнее и в нём легче ошибиться, а на больших данных ещё и заметно тормозит. Как правильно: прежде чем писать цикл, спроси себя «нет ли готового метода или векторной операции?» — почти всегда есть.
  • Ошибка: считать показатели, не посмотрев на данные. Почему: в файле бывают пустые ячейки, текст вместо чисел, лишние пробелы — и расчёт молча даст бессмыслицу, а не ошибку. Как правильно: перед расчётами вызови df.head() и df.columns, убедись, что данные те, что ты думаешь.

7. Памятка: работа с таблицей в Pandas за 6 шагов

Короткий чек-лист, который стоит довести до автоматизма:
  1. Импорты в начале файла: import pandas as pd, при необходимости import numpy as np.
  2. Загрузи данные: df = pd.read_csv("файл.csv") (или собери из словаря через pd.DataFrame).
  3. Осмотрись: df.head() — как выглядят строки; df.columns — точные имена столбцов.
  4. Считай по столбцу: df["поле"].mean(), .sum(), .max(), .min(), .count().
  5. Фильтруй строки: df[df["поле"] > значение] — оставить нужные записи.
  6. Проверь результат здравым смыслом: среднее не больше максимума, число строк правдоподобно, сумма похожа на правду. Не сходится — ищи ошибку, не сдавай.

8. Краткие итоги

  • Библиотека — это готовый проверенный код; import ... as ... подключает его. Стандартные псевдонимы — np и pd.
  • NumPy — про числа и скорость: массив np.array([...]), операции применяются ко всем элементам сразу (векторные операции), без цикла.
  • Pandas — про таблицы: DataFrame = столбцы (по именам) + строки (по индексу).
  • Доступ к столбцу — df["имя"]; возвращает Series, к которому применимы mean(), sum(), max(), min(), count().
  • Фильтрация строк — df[df["поле"] > значение]: условие строит маску из True/False, по ней отбираются строки (читай в два шага).
  • Читай не только свой код, но и ответ библиотеки: там результат, тип данных или готовая подсказка по ошибке.
  • Главные ошибки новичка — забытые импорты (NameError) и неверный регистр имени столбца (KeyError); лечатся проверкой через df.columns.

10. Глоссарий

  • DataFrame — главный объект Pandas: таблица с именованными столбцами и пронумерованными строками.
  • Index (индекс) — подписи строк DataFrame (по умолчанию числа 0, 1, 2, …).
  • Библиотека — готовый набор кода, подключаемый к программе командой import.
  • Булева индексация (фильтрация) — отбор строк таблицы по логическому условию (маске).
  • Векторная операция — действие, применяемое сразу ко всему массиву/столбцу, без явного цикла.
  • Маска — столбец из значений True/False, задающий, какие строки оставить при фильтрации.
  • Массив (ndarray) — основной объект NumPy: набор чисел одного типа с быстрыми операциями над всеми элементами.
  • Метод — функция, вызываемая у объекта через точку (df["cena"].mean()).
  • NumPy — библиотека для быстрых вычислений над массивами чисел.
  • Pandas — библиотека для работы с табличными данными.
  • Псевдоним (alias) — короткое имя библиотеки при импорте (numpy → np).
  • Series — один столбец DataFrame; набор значений с индексом и методами расчёта.
