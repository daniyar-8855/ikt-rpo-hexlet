### Пара 1. Сломанный код: запусти, прочитай traceback, найди причину

**Цель:** научиться чинить код по сообщению об ошибке, а не «методом тыка».

**Задача 1. IndexError.** Создай файл `summator.py` со следующим (сломанным) кодом и запусти `python summator.py`:

```python
def sum_list(nums):
    total = 0
    # ошибка где-то здесь
    for i in range(len(nums) + 1):
        total += nums[i]
    return total

print(sum_list([10, 20, 30]))
```

Программа упадёт с ошибкой вида:

```
Traceback (most recent call last):
  File "summator.py", line 8, in <module>
    print(sum_list([10, 20, 30]))
  File "summator.py", line 5, in sum_list
    total += nums[i]
IndexError: list index out of range
```

**Что делаешь.** Прочитай traceback: тип ошибки (`IndexError`), строку (`total += nums[i]`) и причину. Список из 3 элементов имеет индексы `0, 1, 2`, а `range(len(nums) + 1)` даёт `0, 1, 2, 3` — на индексе `3` элемента нет. Исправь: убери `+ 1` (правильно `range(len(nums))`) или, проще и надёжнее, перебирай сами элементы:

```python
def sum_list(nums):
    total = 0
    for x in nums:
        total += x
    return total
```

Запусти снова — должно вывести `60`.

**Задача 2. ZeroDivisionError.** Создай файл `srednee.py` и запусти его:

```python
def average(numbers):
    total = sum(numbers)
    return total / len(numbers)

print(average([2, 4, 6]))   # выведет 4.0
print(average([]))          # здесь упадёт
```

Ошибка:

```
Traceback (most recent call last):
  File "srednee.py", line 6, in <module>
    print(average([]))
  File "srednee.py", line 3, in average
    return total / len(numbers)
ZeroDivisionError: division by zero
```

**Что делаешь.** По traceback видно: на пустом списке `len(numbers)` равен `0`, происходит деление на ноль. Исправь, обработав крайний случай явно (не «глотая» ошибку через пустой `try/except`):

```python
def average(numbers):
    if not numbers:
        return 0          # или: raise ValueError("список пуст")
    return sum(numbers) / len(numbers)
```

Запусти снова — на `[2, 4, 6]` вернёт `4.0`, на `[]` вернёт `0` (или осмысленную ошибку, если выбрал `raise`).

**Если причина непонятна** — сформулируй запрос к ИИ: вставь **текст ошибки и фрагмент кода**, попроси объяснить причину и предложить исправление, **не маскируя баг**. Не вставляй пароли, ключи, реальные данные. Проверь гипотезу ИИ сам: запусти исправленный код.

**Критерии приёмки (Пара 1):**

- оба файла **запущены**, traceback прочитан; в тетради/отчёте записан тип ошибки и строка для каждой задачи;
- **своими словами** объяснена причина каждой ошибки (выход за границы списка; деление на ноль на пустом вводе);
- обе ошибки исправлены, исправленный код **запущен** и даёт верный результат (`60`; `4.0` и обработанный пустой список);
- если использовался ИИ — указано, для чего именно (объяснение ошибки / идея исправления), и отмечено, что гипотеза проверена запуском;
- **устная защита:** ты можешь показать строку traceback и объяснить, почему код падал и почему твоя правка устраняет причину, а не маскирует её.

### Пара 2. Тесты на pytest для своих функций

**Цель:** закрыть исправленный код тестами, включая граничные случаи, и убедиться, что тесты проходят.

**Что делаешь.** Создай два файла в одной папке.

Файл `main.py` (рабочие функции — можно взять исправленные из Пары 1):

```python
def average(numbers):
    if not numbers:
        return 0
    return sum(numbers) / len(numbers)


def count_vowels(text):
    vowels = "аеёиоуыэюяАЕЁИОУЫЭЮЯ"
    count = 0
    for ch in text:
        if ch in vowels:
            count += 1
    return count


def is_even(n):
    return n % 2 == 0
```

Файл `test_main.py` (тесты; обрати внимание на строку `from main import ...`):

```python
from main import average, count_vowels, is_even


def test_average_normal():
    assert average([2, 4, 6]) == 4          # обычный случай


def test_average_single():
    assert average([5]) == 5                # один элемент


def test_average_empty():
    assert average([]) == 0                 # крайний: пустой список


def test_count_vowels_normal():
    assert count_vowels("окно") == 2        # о, о


def test_count_vowels_empty():
    assert count_vowels("") == 0            # крайний: пустая строка


def test_is_even_true():
    assert is_even(4) is True


def test_is_even_false():
    assert is_even(7) is False
```

**Как запустить.** Открой терминал в папке с этими файлами и выполни:

```
pytest
```

(или `pytest -v` — подробнее, по каждому тесту). Если pytest не установлен — `pip install pytest`. Ожидаемый результат — все тесты зелёные, внизу строка вида `7 passed`.

**Самостоятельная часть.** Допиши тесты ещё для **2–3 своих функций** (можешь взять `sum_list` из Пары 1). Для каждой функции обязательно добавь **граничный случай** (пустой ввод, ноль, отрицательное число, один элемент). Покажи, что `pytest` выводит `passed` по всем тестам.

> **Подсказка по граничным случаям.** Для списков — пустой `[]` и из одного элемента; для чисел — `0` и отрицательное; для строк — пустая `""`. Именно на них код чаще всего ломается.

**Критерии приёмки (Пара 2):**

- созданы `main.py` и `test_main.py`, тест импортирует функции через `from main import <функция>`;
- покрыты **2–3 функции**, у каждой есть тест на **обычный и граничный** случай;
- команда `pytest` выполнена, на экране/скриншоте видно, что все тесты **проходят** (`passed`);
- ты можешь объяснить, **почему** выбран каждый граничный случай (на чём код мог бы упасть);
- если использовался ИИ для генерации тестов — указано это, и каждый `assert` ты проверил/подобрал сам;
- **устная защита:** преподаватель может попросить добавить ещё один тест «на ходу» (например, на новый граничный случай) — ты дописываешь его и запускаешь `pytest`.

> **AI-устойчивость.** ИИ может предложить тест с **неверным ожидаемым значением** в `assert` — поэтому каждое значение проверяй сам, запуская код. На защите нужно уметь объяснить логику теста, а не просто показать зелёный экран.
