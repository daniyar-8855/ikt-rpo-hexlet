# Стандарт оформления занятия v2 (по эталону НВТП)

Этот документ — спецификация для обновления занятий курса ИКТ-РПО до стандарта v2.
Эталон (образец для подражания): занятие **33_основы-ии-агентов** в этой же папке.

## Что обновляем (4 направления). НЕ трогаем upr.md и zadanie.html.

Файлы занятия после обновления: `urok.md`, `urok.html`, `test.gift`, `plan.md`, **`plan.html`** (новый), `upr.md` (без изменений), `zadanie.html` (без изменений), папка **`assets/`** с 1–2 SVG-схемами.

### 1. urok.md и urok.html — педагогическая структура

Порядок разделов (заголовки строго такие):

1. `# Заголовок темы`
2. `## Практическая ситуация` — короткий хук из реальной жизни/профессии (2 абзаца). Сразу после — встроить ПЕРВУЮ схему.
3. `## Что ты научишься делать` — маркированный список (3–4 пункта, глаголами).
4. `## Почему это важно` — мотивация + абзац «Связь с профессией: …».
5. `## Учимся читать схему` — отсылка к схеме + 3 вопроса по ней.
6. `## Главное понятие` — ключевой термин. В urok.md как блок `> **Термин** — определение.`; в urok.html в callout info.
7. Содержательные разделы темы (сохранить из старого урока факты/таблицы/код). Вторую схему встроить в подходящий раздел.
8. `### Мини-кейс` — короткий разбор примера.
9. `## Разбор типичной ошибки` — формат: **Ошибка.** / **Почему это ошибка.** / **Как правильно.** (в urok.html — в красном callout danger).
10. `## Практика` — 2 письменных задания + блок **Образец (часть ответа…)** (в urok.html — info callout).
11. `## Самопроверка` — 3 пункта-чеклиста «Я знаю/умею …».
12. `## Подумай` — 2 рефлексивных вопроса (связь с профессией/жизнью).
13. `## Итог` — список ключевых выводов (в urok.html — зелёный callout success «Что теперь делать»).
14. `## Полезные ссылки` — 3–5 ссылок (сохранить из старого урока).
15. Через `---`: строка `*Источник: …*` (учебники главами/рамки — БЕЗ номеров страниц; для ИИ-тем — DigComp 2.2 / UNESCO AI Framework 2024 / офиц. документация), затем строка грифа.

Объём теории 600–900 слов. Язык: русский, на «ты», без эмодзи.

ГРИФ (дословно, последняя строка):
`*Материал разработан рабочей группой ТОО «Колледж Хекслет Казахстан» и одобрен к использованию в обучении решением Педагогического совета.*`
В urok.html гриф: `<p style="font-size:0.85em;color:#666"><em>…</em></p>` (и отдельной такой же строкой — Источник).

### Callout-блоки в urok.html (точные цвета)
- info: `background:#e8f4fd;border-left:4px solid #2196F3`
- warning: `background:#fff8e1;border-left:4px solid #FFC107`
- danger: `background:#fdecea;border-left:4px solid #f44336`
- success: `background:#e8f5e9;border-left:4px solid #4CAF50`
Шаблон: `<div style="background:#e8f4fd;border-left:4px solid #2196F3;padding:12px 16px;margin:16px 0;border-radius:4px">…</div>`
Между крупными разделами — `<hr>`. Код — в `<pre><code>…</code></pre>`. Картинка: `<p style="margin:16px 0"><img src="assets/ИМЯ.svg" alt="…" style="max-width:100%;height:auto"></p>`.

### 2. SVG-схемы (папка assets/)

1–2 схемы под тему (структура / цикл / сравнение / поток данных / классификация). Имя файла: `NN_shema-краткое-описание.svg` (NN — номер занятия, латиницей-транслитом).

Стиль (как в эталоне):
- `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 760 ВЫСОТА" font-family="Arial, sans-serif">`
- первый элемент — белый фон: `<rect x="0" y="0" width="760" height="ВЫСОТА" fill="#ffffff"/>`
- заголовок схемы: `font-size="18" font-weight="bold" fill="#1a3c5e"`, по центру (x=380)
- блоки: прямоугольники `rx="8"` — акцентные `fill="#2196F3"` с белым текстом; обычные `fill="#e8f4fd" stroke="#2196F3"`; «правильные/итог» `fill="#e8f5e9" stroke="#4CAF50"`
- текст в блоках `font-size="12–14"`, тёмный `fill="#0d2b45"` / `#1b5e20`
- стрелки/линии `stroke="#90caf9"`; при необходимости `<marker>` для наконечников
- ВЕСЬ текст — на русском, кратко, помещается в блоки (следи за шириной).

### 3. test.gift — богаче

Сохранить $CATEGORY и 12 вопросов (банк), формат GIFT, UTF-8. Добавить:
- обратную связь на варианты: `=верный # Верно: пояснение.` и `~неверный # почему неверно`;
- 1–2 вопроса верно/неверно: `::NN_03:: Утверждение. {TRUE #### пояснение}` или `{FALSE #### пояснение}`;
- 1–2 встроенных пропуска в предложении: `Текст {=ответ #### пояснение} продолжение.`;
- 1 shortanswer с несколькими допустимыми ответами в конце.
НЕ использовать matching (MCP при сборке его не поддерживает).
Комментарий-шапка: `// Тест к занятию NN «…». GIFT, UTF-8. Банк 12 → 10.`

### 4. plan.md и plan.html — официальная форма (приказ МОН РК № 130, прил. 8)

Структура (как в эталоне 33):
- Шапка: Организация (ТОО «Колледж Хекслет Казахстан») / Дисциплина (Применение информационно-коммуникационных и цифровых технологий) / Тема занятия / Подготовил педагог: ____ / Дата: «__» ___ 20__ г.
- `## 1. Общие сведения` — Курс, группа (квалификация 4S06130103 «Разработчик ПО»); Тип занятия; Продолжительность (по часам занятия: 2 ч=90 мин/1 пара, 4 ч=2 пары, 6 ч=3 пары); строка «Результат обучения: РО 2.x · Связь с ПМ: …».

- `## 3. Ожидаемые результаты` — «Обучающийся: …» список.
