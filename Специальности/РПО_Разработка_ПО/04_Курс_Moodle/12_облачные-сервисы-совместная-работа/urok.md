# Облачные сервисы и совместная работа

Файл «итоговый_финал_2_правка.docx» в пяти версиях по почте — это боль. В команде так работать нельзя. Облачные сервисы решают проблему: один документ, общий доступ, совместное редактирование, история версий. Для разработчика облако — это и хранилище, и среда командной работы, и (в перспективе) место, где живёт его приложение. Этот урок — про грамотную работу с облаком.

## Чему ты научишься

- объяснять, что такое облачные технологии и зачем они команде;
- настраивать общий доступ и совместное редактирование без хаоса;
- понимать риски облака и базовые правила безопасности.

## Что такое облако

**Облачные технологии** — хранение и обработка данных на удалённых серверах с доступом через интернет. Модели по уровню готовности:

- **Хранилище** (Google Drive, OneDrive, Yandex Disk) — файлы и совместная работа.
- **Сервисы-приложения** (Google Docs, Sheets) — работа прямо в браузере.
- **Инфраструктура для разработчиков** (облачные серверы, БД) — где запускают приложения.

## Совместная работа без хаоса

- **Один источник правды:** один файл/папка с общим доступом, а не копии по почте.
- **Уровни доступа:** «просмотр», «комментирование», «редактирование» — давай минимально необходимый.
- **История версий:** можно откатиться к любой прошлой версии — не бойся редактировать.
- **Комментарии и задачи** прямо в документе вместо переписки.

### Мини-кейс
Команда вела отчёт в общем Google-документе. Один участник «всё удалил». Паника? Нет: открыли историю версий и восстановили нужное состояние за минуту. Следующий шаг: настроить право «комментирование» для тех, кто не должен править напрямую.

## Риски и безопасность

- **Доступ по ссылке «всем, у кого есть ссылка»** = публикация. Давай доступ конкретным людям.
- **Конфиденциальные данные** не клади в личное облако без согласования.
- **2FA** на облачный аккаунт обязательна — там вся работа.
- Помни: файл в облаке доступен, пока есть интернет и аккаунт активен.

## Частые ошибки

**Ошибка:** «расшарить» документ с данными ссылкой «для всех».
**Почему:** ссылка утечёт — данные станут публичными.
**Как правильно:** доступ по приглашению конкретным аккаунтам.

**Ошибка:** плодить копии файла вместо общего доступа.
**Почему:** теряется актуальная версия, рассинхрон в команде.
**Как правильно:** один общий файл + история версий.

## Итог: что теперь делать

- Используй облако как единый источник правды, а не копии по почте.
- Давай минимально необходимый уровень доступа конкретным людям.
- Пользуйся историей версий — не бойся редактировать.
- Включи 2FA и не клади конфиденциальные данные в личное облако.
