# Упражнения. Занятие 19. Открытые данные и API

> Формат — работа с данными и разбор. Критерии приёмки под каждым заданием.

## Упражнение 1. Найди набор данных
Найди на data.egov.kz любой набор по теме (население, образование, экономика). Укажи название, формат и за какой период данные.

**Критерий приёмки:** набор найден; указаны название, формат (CSV/XLSX), период.

## Упражнение 2. CSV или JSON
Для каких задач удобнее CSV, а для каких JSON: (а) открыть в Excel и посмотреть; (б) получить из API в коде?

**Критерий приёмки:** (а) CSV/XLSX; (б) JSON.

## Упражнение 3. Объясни API своими словами
Опиши на примере (курс валют, погода), как программа получает данные через API: что отправляет, что получает.

**Критерий приёмки:** есть запрос к сервису и ответ (обычно JSON); данные обновляются без ручного ввода.

## Упражнение 4. Открытые vs персональные
Раздели на «можно публиковать как открытые данные» / «нельзя»: статистика по городам; ФИО и телефоны жителей; число школ по регионам.

**Критерий приёмки:** можно — статистика по городам и число школ; нельзя — ФИО и телефоны (персональные).

## Упражнение 5. Найди ошибку
Студент построил дашборд на выдуманных числах. Почему это проблема и как исправить?

**Критерий приёмки:** недостоверность; взять реальный набор с data.egov.kz.

## СРС (самостоятельная работа, 2 ч)

**Тема:** Выгрузка и анализ датасета с data.egov.kz

Выгрузи датасет с data.egov.kz и сделай 2–3 простых вывода. Используй обезличенные/тестовые данные; если применял ИИ — укажи, для чего именно.

**Критерий приёмки:** датасет выгружен, есть выводы, данные обезличены. Работа сдаётся преподавателю (файл/отчёт) и засчитывается как СРС по теме.
