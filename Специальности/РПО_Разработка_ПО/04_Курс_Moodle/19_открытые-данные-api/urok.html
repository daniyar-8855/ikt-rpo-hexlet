Порталы открытых данных (data.egov.kz) и API

Где взять реальные данные для проекта, дашборда или анализа? Не выдумывать, а брать из открытых источников. Государство публикует наборы на data.egov.kz, а многие сервисы отдают данные через API. Уметь находить открытые данные и понимать API — база для работы с данными и интеграций.

Чему ты научишься: находить и скачивать открытые данные; понимать, что такое API; различать форматы (CSV, JSON).

Открытые данные

Открытые данные — наборы, опубликованные свободно для повторного использования. На data.egov.kz — статистика, справочники, показатели по регионам и отраслям (часто CSV/XLSX).

Зачем разработчику: реальные данные для дашбордов, прототипов, учебных проектов — без нарушения приватности (данные обезличены).

Что такое API

API — способ одной программы запрашивать данные/функции у другой. Программа делает запрос и получает ответ (обычно JSON), без ручного скачивания.

Идея: приложение запрашивает курс валют через API банка и показывает актуальные цифры автоматически.

Мини-кейс

Студент «придумал» цифры населения городов для дашборда. Преподаватель попросил реальные. Решение: скачать набор с data.egov.kz (CSV) и построить дашборд на настоящих данных.

Ошибка: использовать выдуманные данные «для вида».
Почему: выводы недостоверны.
Как правильно: брать открытые данные из официальных источников.
Ошибка: «открытые данные = можно публиковать что угодно о людях».
Почему: открытые данные обезличены; персональные так публиковать нельзя.
Как правильно: работать с обезличенными наборами.
Итог — что теперь делать:
  • бери реальные данные с data.egov.kz;
  • понимай API как программный способ получать данные (запрос → ответ JSON);
  • различай форматы: CSV/JSON — для кода, XLSX — для людей;
  • не путай открытые данные с персональными.
