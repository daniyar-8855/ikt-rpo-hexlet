# План занятия 19. Порталы открытых данных (data.egov.kz) и API

> Для преподавателя (в Moodle — скрытая «Страница»). Тип: лабораторно-практическое, 2 ч. РО 2.2.

- **Дисциплина:** Применение ИКТ и цифровых технологий (РПО, 4S06130103)
- **Тема:** Порталы открытых данных (data.egov.kz) и API
- **Цель:** научить находить открытые данные и понимать принцип API.
- **Результат обучения:** РО 2.2 (связь с работой с данными РО 2.3).

## Ожидаемые результаты
Студент умеет: находить и скачивать открытые данные; объяснить, что такое API; различать форматы CSV/JSON/XLSX; не путать открытые и персональные данные.

## Ход занятия (90 мин)
| Этап | Время | Содержание |
|---|---|---|
| Орг. момент | 5 мин | Зацепка: «где взять реальные данные» |
| Теория | 20 мин | Открытые данные; API; форматы |
| Демонстрация | 20 мин | data.egov.kz: поиск и скачивание набора |
| Практика | 30 мин | Упражнения 1–5 (поиск, форматы, API, открытые vs персональные) |
| Выдача задания | 10 мин | Анализ набора данных по вариантам |
| Итог, рефлексия | 5 мин | Что делать / где применяется / как не ошибиться |

## Средства обучения
Проектор, ПК с доступом к data.egov.kz, табличный редактор.

## Контроль
- Текущий: упражнения, тест (банк 12, 10 случайных).
- Задание: анализ открытого набора по вариантам + устная защита.

## Источники
- ГОСО ТиПО (приказ МП РК № 348); data.egov.kz; MDN (API, JSON).

При наличии обучающихся с особыми образовательными потребностями предусматриваются адаптация темпа, дополнительные пояснения и индивидуальные задания, одобренные методическим объединением.
