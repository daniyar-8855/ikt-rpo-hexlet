# Порталы открытых данных (data.egov.kz) и API

Где взять реальные данные для проекта, дашборда или учебного анализа? Не выдумывать, а брать из открытых источников. Государство публикует наборы данных на data.egov.kz, а многие сервисы отдают данные через **API** — программный интерфейс. Уметь находить открытые данные и понимать, что такое API, — база для работы с данными и интеграций.

## Чему ты научишься

- находить и скачивать наборы открытых данных (data.egov.kz);
- понимать, что такое API и зачем он нужен;
- отличать форматы данных (CSV, JSON) и оценивать их пригодность.

## Открытые данные

**Открытые данные** — наборы, опубликованные свободно для повторного использования. На **data.egov.kz** есть статистика, справочники, показатели по регионам и отраслям. Их можно скачать (часто CSV/XLSX) и анализировать.

Зачем разработчику: реальные данные для дашбордов, обучения моделей, прототипов, учебных проектов — без нарушения приватности (данные обезличены и публичны).

## Что такое API

**API (Application Programming Interface)** — способ одной программы запрашивать данные/функции у другой. Вместо того чтобы скачивать файл вручную, программа делает **запрос** и получает **ответ** (обычно в формате JSON).

Пример идеи: твоё приложение запрашивает курс валют через API банка и показывает актуальные цифры — без ручного обновления.

## Форматы данных

| Формат | Что это | Удобство |
|---|---|---|
| CSV | таблица, значения через запятую | просто, открывается в Excel |
| JSON | структура «ключ–значение» | удобно для API и кода |
| XLSX | таблица Excel | для людей, не для кода |

### Мини-кейс
Студент для учебного дашборда «придумал» цифры населения городов. Преподаватель попросил реальные. Следующий шаг: скачать набор с data.egov.kz (CSV) и построить дашборд на настоящих данных.

## Частые ошибки

**Ошибка:** использовать выдуманные данные в проекте «для вида».
**Почему:** выводы недостоверны, проект несерьёзен.
**Как правильно:** брать открытые данные из официальных источников.

**Ошибка:** считать, что «открытые данные» = «можно публиковать что угодно о людях».
**Почему:** открытые данные обезличены; персональные данные так публиковать нельзя.
**Как правильно:** работать с обезличенными наборами, уважать приватность.

## Итог: что теперь делать

- Бери реальные данные с порталов открытых данных (data.egov.kz).
- Понимай API как способ программно получать данные (запрос → ответ JSON).
- Различай форматы: CSV/JSON — для кода, XLSX — для людей.
- Не путай открытые данные с персональными.
