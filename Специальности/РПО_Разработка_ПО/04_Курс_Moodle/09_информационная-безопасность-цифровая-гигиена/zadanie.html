Задание к занятию 09. Аудит цифровой гигиены

Вариант — по номеру в журнале (свой профиль для аудита на ~5 человек). № 1–5 → В1 · 6–10 → В2 · 11–15 → В3 · 16–20 → В4 · 21–25 → В5 · 26–30 → В6.

Обязательная часть (до 85 баллов)

  1. Проведи аудит цифровой гигиены персонажа своего варианта: найди минимум 5 проблем.
  2. Для каждой проблемы укажи риск и как исправить.
  3. Составь для персонажа план защиты: пароли, 2FA, резервные копии (3-2-1), приватность, секреты.
  4. Отдельно отметь, какие действия нарушают Закон РК «О персональных данных».
Продвинутая часть (+15 баллов): настрой реально (на учебном/тестовом аккаунте) менеджер паролей и 2FA, приложи скриншоты, объясни устно.

Профили для аудита по вариантам

  • В1. Студент: один пароль везде, 2FA нет, файлы на одном ноутбуке.
  • В2. Фрилансер: ключи API в коде на GitHub, пароли в браузере на чужом ПК.
  • В3. Блогер: выкладывает скриншоты с личными данными, нет резервных копий.
  • В4. Команда проекта: общий аккаунт, пароль в чате, реальные данные клиентов в тестах.
  • В5. Студент: «пароли.txt» на рабочем столе, открытые сессии на чужих устройствах.
  • В6. Стажёр: пересылает данные клиентов в публичный чат-бот, не обновляет ПО.

СРС (самостоятельная работа, 2 ч)

Тема СРС: Аудит личной цифровой безопасности (чек-лист).
Задание: Проведи аудит своей цифровой безопасности по чек-листу (пароли, 2FA, обновления, копии). Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: чек-лист заполнен и намечены 3 улучшения.
