Задание к занятию 02. Окружение и структура проекта

Вариант — по номеру в журнале (один тип проекта на ~5 человек). Задачи и рубрика единые.
№ 1–5 → Вариант 1 · № 6–10 → Вариант 2 · № 11–15 → Вариант 3 · № 16–20 → Вариант 4 · № 21–25 → Вариант 5 · № 26–30 → Вариант 6.

Обязательная часть (до 85 баллов)

Для типа проекта своего варианта:

  1. Спроектируй структуру папок проекта (минимум 5 папок/файлов) с правильными именами (латиница, нижний регистр, без пробелов). Покажи деревом.
  2. Объясни выбор ОС для разработки и для сервера (где будет работать продукт) — 2–3 предложения.
  3. Приведи 3 пути внутри проекта: один абсолютный и два относительных (на файлы своего дерева).
  4. Найди и исправь 3 «плохих» имени файла из списка ниже под свой проект:
    Мои Файлы/Главный Скрипт.py, C:\temp\Data.CSV, тест 1.js.
Продвинутая часть (+15 баллов): опиши, как настроишь одинаковое окружение у всех в команде (WSL/контейнер/виртуальная машина) и почему это убирает баг «работает только у меня». Защищается устно.

Типы проектов по вариантам

  • Вариант 1. Веб-сайт колледжа (HTML/CSS/JS, статика + изображения).
  • Вариант 2. Телеграм-бот на Python (код, конфиг, логи, данные).
  • Вариант 3. Мобильное приложение «Расписание» (исходники, ресурсы, тесты).
  • Вариант 4. 2D-игра (код, спрайты/звуки, уровни, сборка).
  • Вариант 5. REST API-сервис (код, миграции БД, тесты, документация).
  • Вариант 6. Проект анализа данных (ноутбуки, датасеты, графики, отчёт).

СРС (самостоятельная работа, 2 ч)

Тема СРС: Сравнительный анализ ОС Windows и Linux.
Задание: Составь сравнительную таблицу Windows и Linux по 5–7 критериям с выводом. Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: в таблице не менее 5 критериев и аргументированный вывод о выборе ОС.
