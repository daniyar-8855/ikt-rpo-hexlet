### Пара 1. Генерация функций и проверка на крайних случаях

Сгенерируй с ИИ-ассистентом **три функции** нарастающей сложности, проверь каждую на крайних случаях и объясни построчно.

**Задание 1.1.** Запроси у ИИ функцию `srednee(chisla)`, которая принимает список чисел и возвращает их среднее арифметическое; без сторонних библиотек. Проверь на крайних случаях: пустой список, один элемент, отрицательные числа.

**Задание 1.2.** Запроси функцию `schitat_glasnye(stroka)`, которая возвращает количество гласных букв (a, e, i, o, u) в строке; без библиотек. Проверь: пустая строка, строка без гласных, строка из одних гласных, заглавные буквы.

**Задание 1.3.** Запроси функцию `bezopasnoe_delenie(a, b)`, которая делит `a` на `b` и корректно ведёт себя при `b == 0` (например, возвращает `None` и сообщает об ошибке). Проверь: обычное деление, деление на ноль, деление отрицательных, деление нуля на число.

Образец того, как должна выглядеть проверенная и понятая функция (твои функции оформляй так же — с docstring, обработкой крайнего случая и тестовыми вызовами):

```python
def srednee(chisla):
    """Возвращает среднее арифметическое списка чисел.
    При пустом списке возвращает None (делить не на что)."""
    if len(chisla) == 0:          # крайний случай: пустой список
        return None
    return sum(chisla) / len(chisla)

# Проверка на крайних случаях:
print(srednee([4, 5, 3, 5, 2]))   # 3.8 — обычный случай
print(srednee([10]))              # 10.0 — один элемент
print(srednee([-2, -4, -6]))      # -4.0 — отрицательные
print(srednee([]))                # None — пустой список, не падает
```

**Критерии приёмки (Пара 1):**
- три рабочие функции, каждая с docstring и обработкой крайнего случая;
- для каждой функции показаны проверки минимум на 3 крайних случаях (пустой ввод, ноль, отрицательные — где применимо);
- ни одна функция не падает на крайнем случае (нет необработанных `ZeroDivisionError`, `IndexError`);
- студент устно объясняет любую строку любой функции своими словами;
- зафиксировано, что выдал ИИ и что студент изменил сам.

### Пара 2. Рефакторинг «грязной» функции и сравнение «до/после»

Возьми рабочую, но «грязную» функцию (ниже), попроси ИИ её улучшить, сравни «до» и «после» и объясни, что и почему изменилось. Главное правило рефакторинга: **поведение функции меняться не должно** — это нужно доказать тестами.

Исходная «грязная», но рабочая функция (скопируй её как есть):

```python
def f(l):
    r = []
    for i in range(len(l)):
        if l[i] % 2 == 0:
            r.append(l[i])
    s = 0
    for j in range(len(r)):
        s = s + r[j]
    return s
```

**Задание 2.1.** Запусти исходную функцию и запиши, что она делает на нескольких входах (например, `[1, 2, 3, 4]`, `[]`, `[1, 3, 5]`, `[-2, -1, 0]`). Сформулируй своими словами назначение функции.

**Задание 2.2.** Попроси ИИ сделать рефакторинг: понятное имя функции и переменных, docstring, проход по элементам вместо `range(len(...))`, по возможности короче. Получи вариант «после».

**Задание 2.3.** Докажи, что поведение не изменилось: прогони «до» и «после» на одних и тех же входах (включая пустой список) и сравни результаты — они обязаны совпасть. Составь короткую таблицу «вход → результат до → результат после».

**Задание 2.4.** Письменно перечисли минимум 3 конкретных улучшения (что было плохо → как стало → почему так лучше).

Образец таблицы сравнения поведения (заполни своими данными):

| Вход | Результат «до» | Результат «после» |

**Критерии приёмки (Пара 2):**
- есть обе версии функции — «до» и «после»;
- доказано совпадение результатов на одинаковых входах (таблица, минимум 4 входа, включая пустой список);
- перечислено не менее 3 улучшений с объяснением «почему так лучше»;
- студент понимает, что рефакторинг не меняет поведения, и может это объяснить;
- зафиксирован запрос к ИИ и какие правки студент внёс поверх ответа ИИ.

### Пара 3. Обработка набора данных с ИИ и устная защита

Реши с ИИ-ассистентом мини-задачу обработки набора данных, проверь корректность вручную и защити решение устно. Данные — **список словарей** (обезличенные/тестовые). Выбери один набор из трёх:

- **товары:** `[{"name": "...", "cena": ...}, ...]`;
- **оценки:** `[{"student": "Студент 1", "ball": ...}, ...]`;
- **логи:** `[{"deystvie": "...", "dlitelnost_sek": ...}, ...]`.

**Задание 3.1.** Создай (или возьми у преподавателя) набор из 6–8 записей. Имена/идентификаторы — обезличенные («Студент 1», «Клиент 2»), никаких реальных ФИО, ИИН, телефонов.

**Задание 3.2.** Поставь ИИ задачу обработать набор: посчитать сумму и среднее по числовому полю, найти запись с максимумом и с минимумом по этому полю, отобрать записи выше среднего. Запрос формулируй по шаблону (язык, вход/выход, ограничения: без сторонних библиотек).

**Задание 3.3.** Проверь решение вручную: возьми 2–3 записи и пересчитай показатель «на бумаге»/в уме — совпадает ли с выводом программы? Отдельно проверь крайний случай: что выведет код на **пустом** списке (не должен падать).

**Задание 3.4.** Найди в наборе «аномалию» (например, цену сильно выше остальных или нулевую длительность) и опиши, как её обнаружить кодом (условие в цикле).

**Задание 3.5.** Подготовься к устной защите: объясни преподавателю любой фрагмент своего решения своими словами и скажи, где именно ты применил ИИ и что проверил сам.

Образец корректной обработки списка словарей (твоё решение строй по этому образцу, поля и условия — свои):

```python
zakazy = [
    {"name": "Клиент 1", "summa": 1500},
    {"name": "Клиент 2", "summa": 2000},
    {"name": "Клиент 3", "summa": 800},
    {"name": "Клиент 4", "summa": 3200},
]

if len(zakazy) == 0:                       # защита от пустого набора
    print("Данных нет")
else:
    summy = [z["summa"] for z in zakazy]   # вытащили числовое поле
    itogo = sum(summy)
    srednee = itogo / len(summy)

    # запись с максимумом по полю "summa"
    maks = zakazy[0]
    for z in zakazy:
        if z["summa"] > maks["summa"]:
            maks = z

    # записи выше среднего
    vyshe_srednego = []
    for z in zakazy:
        if z["summa"] > srednee:
            vyshe_srednego.append(z["name"])

    print("Сумма:", itogo)                  # 7500
    print("Среднее:", srednee)              # 1875.0
    print("Максимум:", maks["name"], maks["summa"])  # Клиент 4 3200
    print("Выше среднего:", vyshe_srednego) # ['Клиент 2', 'Клиент 4']
```

**Критерии приёмки (Пара 3):**
- набор — список словарей из 6–8 обезличенных записей, без реальных персональных данных;
- решение считает сумму, среднее, максимум и минимум, отбирает записи по условию;
- есть защита от пустого списка (на пустом наборе код не падает);
- ручная проверка 2–3 записей совпала с выводом программы;
- найдена и описана аномалия;
- студент прошёл устную защиту: объяснил код своими словами и указал, где применял ИИ и что проверил сам.
