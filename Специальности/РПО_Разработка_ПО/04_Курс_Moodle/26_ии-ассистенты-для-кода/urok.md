# ИИ-ассистенты для генерации и объяснения кода

ИИ-ассистент (ChatGPT, GitHub Copilot, Gemini) стал частью работы разработчика: он пишет черновик функции, объясняет чужой код, подсказывает синтаксис, помогает разобрать ошибку. Это сильно ускоряет работу — но только если ты умеешь им пользоваться и **проверять** результат. Этот урок — про то, как сделать ИИ-ассистента помощником, а не источником багов.

## Чему ты научишься

- использовать ИИ-ассистента для генерации и объяснения кода;
- формулировать запрос так, чтобы получить полезный ответ;
- критически проверять сгенерированный код перед использованием.

## Что умеет ИИ-ассистент

- **Сгенерировать** черновик функции по описанию.
- **Объяснить** незнакомый код построчно.
- **Найти и объяснить ошибку**, предложить исправление.
- **Подсказать** синтаксис, методы, примеры.
- **Переписать** код (рефакторинг), добавить комментарии.

> Ключевая идея: ИИ даёт **черновик**. Финальный код — твоя ответственность.

## Как формулировать запрос

Чем точнее запрос — тем лучше ответ. Указывай:

- **язык и версию** (Python 3, JavaScript);
- **что нужно** (функция, которая делает …);
- **вход и выход** (на входе список чисел, на выходе сумма чётных);
- **ограничения** (без сторонних библиотек).

Плохо: «напиши код». Хорошо: «Напиши на Python функцию, которая принимает список чисел и возвращает сумму чётных; без библиотек; добавь короткие комментарии».

## Проверка сгенерированного кода

Прежде чем использовать:

1. **Пойми**, что делает каждая строка (можешь объяснить — значит понял).
2. **Запусти** и проверь на разных данных, включая крайние случаи.
3. **Проверь безопасность** (нет утечек, инъекций, секретов).
4. **Сверь с документацией**, если есть сомнения.

### Мини-кейс
ИИ сгенерировал функцию деления, которая падает при делении на ноль. Студент сдал без проверки. Следующий шаг: тестировать на крайних случаях (ноль, пустой ввод, отрицательные) и добавить обработку.

## Частые ошибки

**Ошибка:** вставлять сгенерированный код, не читая.
**Почему:** в нём могут быть баги, уязвимости, лишнее.
**Как правильно:** понять код, протестировать, потом использовать.

**Ошибка:** отправлять ИИ реальные данные/секреты в запросе.
**Почему:** нарушение приватности и риск утечки.
**Как правильно:** обезличивать, не передавать ключи и персональные данные.

## Итог: что теперь делать

- Используй ИИ для черновика, объяснения и разбора ошибок.
- Формулируй точный запрос: язык, задача, вход/выход, ограничения.
- Всегда проверяй: понимание → запуск → крайние случаи → безопасность.
- Не передавай ИИ секреты и персональные данные.
