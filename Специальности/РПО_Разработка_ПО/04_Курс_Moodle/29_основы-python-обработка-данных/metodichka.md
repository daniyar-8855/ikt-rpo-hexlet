## 3. Ключевые понятия

- **Контейнер данных** — способ хранения значений в программе (переменная, список, словарь) [3].
- **Переменная** — именованная ячейка, хранящая одно значение.
- **Список (list)** — упорядоченный набор значений, доступ по номеру (индексу) с нуля.

- **Ключ** — имя поля в словаре, по которому достают значение.
- **Цикл `for`** — конструкция, повторяющая действие для каждого элемента набора.
- **Условие `if`** — конструкция, выполняющая блок только при истинности проверки.
- **`append`** — метод, добавляющий элемент в конец списка.
- **Граничный (крайний) случай** — необычные входные данные (пустой список, ноль), на которых код часто ломается.

### 4.1. Контейнеры данных: где живут твои значения

Любая программа работает с данными, а данные надо где-то держать. **Контейнер данных** — это способ хранения значений в памяти. Для повседневных задач в Python хватает трёх контейнеров: переменной, списка и словаря. Посмотри на схему «Контейнеры данных в Python» (приложение А): на ней видно, чем они отличаются наглядно. Разберём каждый на живом коде.

**Переменная — одна ячейка для одного значения.** Самый простой контейнер. Ты даёшь значению имя и дальше обращаешься к нему по этому имени:

```python
cena = 1500
print(cena)        # 1500
cena = cena + 200  # положили в ту же ячейку новое значение
print(cena)        # 1700
```

Что видим в консоли: сначала `1500`, потом `1700`. Почему так — вот тут ловушка, на которой застревает половина новичков со школьной математикой в голове. Знак `=` здесь — **не «равно»**, а **присваивание**: «положи значение справа в ячейку с именем слева». Поэтому строка `cena = cena + 200` — не парадокс и не уравнение, а команда: «возьми текущее значение `cena` (1500), прибавь 200 (стало 1700) и положи результат обратно в `cena`». Читай такие строки всегда справа налево: сначала вычисляется правая часть, потом записывается в левую. Переменная хранит ровно одно значение — одну цену, одно имя, одно число.

**Список (`list`) — упорядоченный набор значений.** Когда значений много и они однотипны (все цены, все температуры, все оценки), их держат в списке. Список пишут в квадратных скобках через запятую:

```python
ceny = [1500, 2000, 1800]
print(ceny)        # [1500, 2000, 1800]
print(len(ceny))   # 3 — в списке три элемента
```

Удобная картинка в голове: список — это один столбец таблицы, где значения идут сверху вниз по порядку. К отдельному элементу обращаются по **индексу** — порядковому номеру в квадратных скобках. И вот первое, что должно засесть намертво: **нумерация в Python идёт с нуля**, а не с единицы.

```python
ceny = [1500, 2000, 1800]
print(ceny[0])     # 1500 — ПЕРВЫЙ элемент, индекс 0
print(ceny[1])     # 2000 — второй элемент
print(ceny[2])     # 1800 — третий (и последний) элемент
```

То есть у списка из трёх элементов индексы — `0`, `1`, `2`. Последний индекс всегда на единицу меньше длины. Запомни это правило, оно спасёт тебе часы: **последний индекс = длина − 1**. Если ты по школьной привычке напишешь `ceny[3]`, программа не промолчит — она упадёт:

```python
print(ceny[3])
# IndexError: list index out of range
```

Читается это как «такого элемента нет, ты вышел за границу списка». Это одна из самых частых ошибок новичка, и берётся она ровно из счёта с нуля. Кстати, полезный приём: последний элемент можно взять как `ceny[-1]` — минус означает «считать с конца», и `-1` это всегда последний, сколько бы элементов ни было. Список, в отличие от переменной, можно менять: добавлять элементы, заменять, удалять — этим он и удобен для накопления результата (увидишь в 4.3).

**Словарь (`dict`) — пары «ключ — значение».** Список хорош, когда значения однородны. Но что делать, если нужно сохранить *одну запись* о товаре, где у каждого поля своё имя — название, цена, количество? Складывать это в список — плохая идея: придётся помнить, что под индексом `0` лежит название, под `1` — цена, под `2` — количество. Через неделю ты сам забудешь этот порядок и всё перепутаешь. Для таких записей есть словарь — он хранит значения не по номеру, а по **ключу** (имени поля). Пишут словарь в фигурных скобках:

```python
tovar = {"name": "Мышь", "cena": 1500}
print(tovar["name"])   # Мышь   — достаём по ключу "name"
print(tovar["cena"])   # 1500   — достаём по ключу "cena"
```

Картинка: если список — это столбец, то словарь — это одна строка таблицы с подписанными полями. Главное отличие от списка — **способ доступа**: к списку обращаемся по номеру `ceny[0]`, к словарю по ключу `tovar["name"]`. Не путай скобки и способ обращения: квадратные скобки используются в обоих случаях, но внутри у списка число, у словаря — ключ в кавычках. Обратишься к словарю по числу (`tovar[0]`) — получишь `KeyError`, потому что ключа `0` в нём нет.

**Как выбрать контейнер — короткое правило на каждый день:**
- одно значение → **переменная** (`cena = 1500`);
- много однотипных значений по порядку → **список** (`ceny = [1500, 2000, 1800]`);
- одна запись с разными подписанными полями → **словарь** (`tovar = {"name": "Мышь", "cena": 1500}`);
- много записей → **список словарей** (об этом в 4.6).

Проверь себя на двух задачах прямо сейчас. (1) Хранить температуры за неделю — это семь однотипных чисел по порядку, значит, **список**. (2) Хранить одну запись о студенте — имя, группа, балл — это разные подписанные поля одной записи, значит, **словарь**. А теперь третья, чуть хитрее: хранить журнал группы, где у каждого студента имя, группа и балл? Это много записей — значит, **список словарей**. Если ты уверенно отвечаешь на такие вопросы, фундамент темы у тебя уже стоит.

### 4.2. Цикл `for`: пройти по каждому элементу

Контейнер выбрали — теперь надо что-то сделать с *каждым* его элементом: сложить, посчитать, проверить. Делать это руками (`ceny[0]`, потом `ceny[1]`, потом `ceny[2]`) — тупик: элементов могут быть тысячи, да и точное их количество ты заранее не знаешь. Для этого есть **цикл** `for`: он сам берёт элементы по одному и повторяет для каждого один и тот же блок кода [11].

Классика жанра — посчитать сумму списка вручную через цикл:

```python
ceny = [1500, 2000, 1800]
itog = 0
for c in ceny:
    itog = itog + c
print(itog)   # 5300
```

Разберём построчно — здесь видна вся механика цикла, и понять её надо один раз и навсегда:

1. `ceny = [1500, 2000, 1800]` — создали список цен.
2. `itog = 0` — завели переменную-накопитель и обнулили её. Это приём, который ты будешь повторять постоянно: прежде чем что-то накапливать, заводим «копилку» и кладём в неё ноль. Забудешь эту строку — получишь `NameError`, потому что переменной ещё нет.
3. `for c in ceny:` — начало цикла. Читается вслух так: «для каждого элемента `c` из списка `ceny`». Переменная `c` на каждом шаге по очереди становится очередным элементом списка. Двоеточие в конце обязательно.
4. `    itog = itog + c` — тело цикла. Обрати внимание на **отступ в 4 пробела** — он не для красоты, а обязателен: по нему Python понимает, что эта строка относится к циклу. На каждом шаге прибавляем текущий элемент к накопителю.
5. `print(itog)` — эта строка **без отступа**, значит, она вне цикла и выполнится один раз, после того как цикл закончит весь проход.

Что происходит по шагам — проследи глазами значения, это важнее заучивания:

| Шаг | `c` (текущий элемент) | `itog` до | `itog` после |
|---|---|---|---|
| 1 | 1500 | 0 | 1500 |
| 2 | 2000 | 1500 | 3500 |
| 3 | 1800 | 3500 | 5300 |

После третьего шага элементы кончились, цикл завершился, и `print` вывел `5300`. Вот так цикл «проходит по списку»: `c` по очереди становится каждым элементом, а тело выполняется столько раз, сколько элементов в списке. Проверь сумму сам: 1500 + 2000 + 1800 = 5300. Сходится.

**Про отступы — отдельно, потому что на этом спотыкаются все без исключения.** В Python отступ (4 пробела) — это часть синтаксиса, а не оформление. Именно отступом обозначается, какие строки входят в тело цикла. Сравни две почти одинаковые программы:

```python
# Вариант А: print ВНУТРИ цикла (с отступом) — сработает на каждом шаге
for c in [1500, 2000, 1800]:
    print(c)      # выведет 1500, потом 2000, потом 1800 — три строки

# Вариант Б: print ВНЕ цикла (без отступа) — сработает один раз
itog = 0
for c in [1500, 2000, 1800]:
    itog = itog + c
print(itog)       # выведет 5300 — одну строку, после цикла
```

Разница ровно в одном отступе — а поведение программы разное. Поменяешь отступ — поменяется результат. И ещё: увидишь ошибку `IndentationError` — знай, дело в отступах (чаще всего смешал пробелы и табы или забыл отступ после двоеточия).

**Зачем уметь писать сумму вручную, если есть готовый `sum`?** Справедливый вопрос. Готовая функция `sum` (раздел 4.4), конечно, короче и удобнее. Но ручной цикл — это универсальный шаблон «накопителя», который работает не только для суммы: точно так же считают количество подходящих элементов, ищут максимум, собирают новый список. Поняв цикл на простом примере суммы, ты поймёшь и все остальные задачи на нём. `sum` — частный случай, цикл — общий инструмент.

### 4.3. Условие `if`: отобрать только нужное

Часто нужно не обработать все элементы, а отобрать подходящие: цены дороже 1800, температуры выше нуля, отличников из списка оценок. Для отбора служит **условие** `if`: оно выполняет блок кода только тогда, когда проверка истинна. Если проверка ложна — блок просто пропускается.

Самое частое сочетание во всей обработке данных — цикл и условие вместе: проходим по списку и складываем подходящие элементы в новый список. Посмотри на схему «Цикл for + условие if» (приложение Б) — она показывает эту логику как блок-схему. А вот код:

```python
ceny = [1500, 2000, 1800]
dorogie = []
for c in ceny:
    if c >= 1800:
        dorogie.append(c)
print(dorogie)   # [2000, 1800]
```

Разбор построчно:

1. `ceny = [1500, 2000, 1800]` — исходный список.
2. `dorogie = []` — создали **пустой список** для результата. Это та же «копилка», что и в 4.2, но копим в неё не число, а элементы.
3. `for c in ceny:` — проходим по каждой цене.
4. `    if c >= 1800:` — проверяем условие «цена больше или равна 1800». Обрати внимание на **двойной отступ** у следующей строки: `if` вложен в `for`, а тело `if` вложено в `if`. Два уровня вложенности — восемь пробелов.
5. `        dorogie.append(c)` — если условие истинно, добавляем элемент в результат методом `append`. Если ложно — эта строка не выполняется, элемент молча пропускается.
6. `print(dorogie)` — выводим собранный список.

Проследим по шагам, что попадает в результат:

| Шаг | `c` | `c >= 1800`? | Действие | `dorogie` после |
|---|---|---|---|---|
| 1 | 1500 | Нет (ложь) | пропустить | `[]` |
| 2 | 2000 | Да (истина) | append | `[2000]` |
| 3 | 1800 | Да (`>=`, значит истина) | append | `[2000, 1800]` |

Обрати внимание на тонкость третьего шага: `1800 >= 1800` — истина, потому что в операторе `>=` есть «или равно». А теперь смотри, как один символ меняет ответ. Поставь строгое `>` вместо `>=`:

```python
ceny = [1500, 2000, 1800]
dorogie = []
for c in ceny:
    if c > 1800:          # строго больше, без "или равно"
        dorogie.append(c)
print(dorogie)   # [2000]  — 1800 уже НЕ прошло!
```

Теперь `1800 > 1800` — ложь, и в результате осталось только `[2000]`. Один символ поменял результат — вот почему условие всегда читаешь внимательно, а на «дороже 1800» первым делом уточняешь у заказчика: «строго дороже или включая 1800?». Это не занудство, это защита от переделки.

**Операторы сравнения**, которые ты будешь ставить в `if`:

| Оператор | Значение | Пример | Результат |
|---|---|---|---|
| `==` | равно | `o == 5` | истина, если `o` равно 5 |
| `!=` | не равно | `o != 5` | истина, если `o` не 5 |
| `>` | больше | `c > 1800` | строго больше |
| `>=` | больше или равно | `c >= 1800` | больше или равно |
| `<` | меньше | `t < 0` | строго меньше |
| `<=` | меньше или равно | `b <= 60` | меньше или равно |

Отдельно и жирным: **не путай `=` и `==`**. Одиночное `=` — присваивание («положи значение»), двойное `==` — сравнение («равны ли?»). В условии `if` всегда сравнение, поэтому `if o == 5`, а не `if o = 5`. Второе — синтаксическая ошибка (`SyntaxError`), и хорошо, что ошибка: Python не даст тебе случайно перезаписать переменную там, где ты хотел её проверить.

**Шаблон «фильтр».** Запомни эту конструкцию как готовый рабочий шаблон — ты будешь применять его чаще всего остального:

```python
rezultat = []                 # 1. пустой список для результата
for element in ishodnyy:      # 2. проходим по исходному списку
    if element_podhodit:      # 3. проверяем условие
        rezultat.append(element)  # 4. подходящие — в результат
```

Подставь сюда своё условие — и отфильтруешь что угодно: дни выше нуля, задачи дольше 30 минут, совершеннолетних участников, товары со скидкой. Четыре строки, которые закрывают огромный класс задач. Выучи их как таблицу умножения.

### 4.4. Простые показатели: сумму, среднее, максимум считаем готовыми функциями

Складывать вручную циклом полезно для понимания механики, но в реальной работе для базовых показателей в Python уже есть **готовые встроенные функции** [11; 12]. Они короче, быстрее и надёжнее ручного кода — незачем изобретать то, что уже написано и отлажено миллионами людей:

```python
ceny = [1500, 2000, 1800]
print(sum(ceny))             # 5300 — сумма всех элементов
print(len(ceny))             # 3 — количество элементов
print(sum(ceny) / len(ceny)) # 1766.6666666666667 — среднее (сумма / количество)
print(max(ceny))             # 2000 — максимум
print(min(ceny))             # 1500 — минимум
```

Разберём каждую и заодно объясним вывод:

- **`sum(ceny)`** — складывает все числа списка. Заменяет весь ручной цикл-накопитель из 4.2 одной строкой. Здесь 1500 + 2000 + 1800 = `5300`.
- **`len(ceny)`** — возвращает количество элементов (длину). Работает не только со списками, но и со строками, и со словарями. Здесь `3`.
- **Среднее** отдельной функции не имеет — считают как `sum / len`, сумму делят на количество. Здесь `5300 / 3 = 1766.666...`. Заметь длинный «хвост» после точки: деление `/` в Python всегда даёт дробное число, даже если делишь нацело. Это нормально, для отчёта потом округлишь.
- **`max(ceny)`** и **`min(ceny)`** — наибольший и наименьший элемент. Здесь `2000` и `1500`.

Эти пять операций — сумма, количество, среднее, максимум, минимум — и есть основа простой аналитики. Добавь к ним фильтр из 4.3, и ты уже отвечаешь на типичные рабочие вопросы: «сколько всего», «в среднем», «самый дорогой/дешёвый», «сколько подходит под условие».

**Когда нужен счётчик с условием.** Иногда надо посчитать не все элементы, а только подходящие — например, сколько в группе отличников. Готового `len` тут мало: он посчитает всех, а нам нужны только пятёрки. Тогда возвращаемся к ручному циклу, но накапливаем не сумму, а счётчик:

```python
ball = [4, 5, 3, 5, 2]
otlichniki = 0               # счётчик отличников
for o in ball:
    if o == 5:               # условие: оценка ровно 5
        otlichniki = otlichniki + 1   # нашли пятёрку — +1
print(otlichniki)            # 2
```

В консоли `2` — потому что пятёрок в списке ровно две. Здесь наглядно видно, как соединяются три блока сразу: цикл (`for`) проходит по оценкам, условие (`if`) отбирает пятёрки, а накопитель (`otlichniki`) считает их количество. Это и есть то самое «собрать программу из строительных блоков».

**Мини-кейс целиком.** Дан список оценок группы `[4, 5, 3, 5, 2]`. Нужны средний балл и число отличников. Средний: `sum / len = 19 / 5 = 3.8` (проверь: 4+5+3+5+2 = 19, делим на 5 — выходит 3.8). Отличников считаем циклом с условием `if o == 5` — их 2. Одна и та же горстка блоков (контейнер, цикл, условие, функции) закрывает обе задачи. Не надо новых знаний — надо по-разному соединить то, что уже есть.

### 4.5. Защита от пустого списка: самая частая «падающая» ошибка

Теперь — про надёжность кода, без которой программа красиво работает на учебном примере и с треском падает на реальных данных. Посмотри на безобидную с виду строку:

```python
spisok = []                      # вдруг данных не пришло — список пустой
print(sum(spisok) / len(spisok)) # ОШИБКА!
```

Что произойдёт при запуске:

```python
# ZeroDivisionError: division by zero
```

Разберём почему. `sum([])` равно `0` (складывать нечего), `len([])` тоже равно `0` (элементов ноль), и получается `0 / 0` — **деление на ноль**. Математика такого не позволяет, и Python тоже: он останавливает программу с ошибкой `ZeroDivisionError`. На учебном списке из трёх цен такого никогда не случится — там всегда три элемента. Но в реальной программе список приходит откуда-то снаружи: из файла, из базы, от пользователя, от API. И он **вполне может оказаться пустым**: файл пуст, запрос ничего не вернул, пользователь ничего не ввёл. Не предусмотрел — программа упала прямо у клиента, в самый неудобный момент.

**Как правильно — проверять перед делением:**

```python
spisok = []
if len(spisok) > 0:                       # есть ли вообще данные?
    srednee = sum(spisok) / len(spisok)
    print("Среднее:", srednee)
else:
    print("Список пуст — считать среднее не из чего")
```

На пустом списке этот код выведет `Список пуст — считать среднее не из чего` вместо падения. Логика простая: `if len(spisok) > 0` означает «если в списке есть хотя бы один элемент». Только в этом случае считаем среднее. Иначе (`else`) — выводим понятное сообщение. Это называется **защитой от граничного случая**: ты заранее продумываешь необычные входные данные (пустой список) и обрабатываешь их аккуратно, а не надеешься, что «данные всегда придут нормальные». Не придут. Закон подлости в разработке работает безотказно.

Тот же принцип — при обращении по индексу. Прежде чем взять `spisok[0]`, убедись, что список не пуст, иначе получишь уже знакомый `IndexError`. Профессиональная привычка, которая отличает рабочий код от учебного: всегда спрашивать себя «а что, если данных нет / их ноль / их слишком много / там не число, а мусор?» — и закрывать эти случаи заранее. Новичок пишет код для «хорошего» случая; профи пишет и для «плохого».

И снова не путай контейнеры в этих проверках: к списку обращаются по индексу `spisok[0]`, к словарю — по ключу `tovar["name"]`. Перепутаешь — получишь ошибку на ровном месте.

### 4.6. Собираем всё вместе: программа обработки данных целиком

Теперь объединим все пять блоков в одну осмысленную программу — ровно ту, с которой мы начали во введении. Задача: дан список цен заказов, посчитать показатели, отобрать заказы дороже среднего и не упасть на пустых данных.

```python
zakazy = [1500, 2000, 1800, 3200, 950]

if len(zakazy) > 0:                       # защита от пустого списка
    itogo = sum(zakazy)                   # сумма
    kolichestvo = len(zakazy)             # количество
    srednee = itogo / kolichestvo         # среднее
    samyy_dorogoy = max(zakazy)           # максимум
    samyy_deshyovyy = min(zakazy)         # минимум

    dorogie = []                          # фильтр: дороже среднего
    for c in zakazy:
        if c > srednee:
            dorogie.append(c)

    print("Сумма заказов:", itogo)         # 9450
    print("Количество:", kolichestvo)      # 5
    print("Средний чек:", srednee)         # 1890.0
    print("Самый дорогой:", samyy_dorogoy) # 3200
    print("Самый дешёвый:", samyy_deshyovyy) # 950
    print("Дороже среднего:", dorogie)     # [2000, 3200]
else:
    print("Данных нет")
```

Проверим вывод руками, чтобы ты видел — цифры не с потолка. Сумма: 1500 + 2000 + 1800 + 3200 + 950 = `9450`. Количество: `5`. Средний чек: 9450 / 5 = `1890.0`. Максимум `3200`, минимум `950`. Дороже среднего (строго больше 1890): 2000 и 3200 проходят, а 1800, 1500 и 950 — нет. Значит, `dorogie` = `[2000, 3200]`. Всё сходится.

Пройдись по программе глазами и узнай каждый блок: контейнер (`zakazy` — список), защита (`if len > 0`), показатели (`sum`, `len`, среднее, `max`, `min`), фильтр (`for` + `if` + `append`), вывод (`print` с подписями). Это и есть типичная «обработка данных» одним куском: прочитать → посчитать → отфильтровать → свести → показать. Меняешь данные и условие — получаешь решение под свою задачу, а каркас остаётся тот же.

**Шаг к словарям: список записей.** Пока мы считали по «голым» числам. В реальности у данных есть поля: у товара — название и цена, у студента — имя и балл. Тогда данные хранят как **список словарей** — много записей, у каждой подписанные поля:

```python
tovary = [
    {"name": "Мышь", "cena": 1500},
    {"name": "Клавиатура", "cena": 2000},
    {"name": "Коврик", "cena": 800},
]

for t in tovary:
    print(t["name"], "-", t["cena"], "тг")  # достаём поля по ключу
```

Вывод:

```python
# Мышь - 1500 тг
# Клавиатура - 2000 тг
# Коврик - 800 тг
```

А теперь посчитаем среднюю цену по полю `"cena"`. Логика та же, что и раньше, только значение достаём из словаря по ключу:

```python
# средняя цена по полю "cena"
ceny = []
for t in tovary:
    ceny.append(t["cena"])         # собрали цены в отдельный список
print(sum(ceny) / len(ceny))       # 1433.3333333333333
```

Проверь: (1500 + 2000 + 800) / 3 = 4300 / 3 = `1433.33...`. Здесь цикл проходит по списку записей, а внутри мы достаём нужное поле по ключу (`t["cena"]`) и складываем цены в отдельный список — дальше уже знакомые `sum` и `len`. Так устроены почти все реальные данные — записи из базы, ответы API, строки таблицы. С этого начинается следующая тема — pandas (занятие 30), где такие таблицы обрабатываются ещё короче, буквально в одну строку. Уэс Маккинни в книге «Python и анализ данных» (главы о структурах данных) прямо показывает: весь анализ строится ровно на этих базовых контейнерах и операциях — ты уже владеешь их основой [2].

## 5. Разобранные примеры и мини-кейсы

**Пример 1 — фильтр температур.** Дан список температур `[12, 18, 9, 21, 15]`. Нужно оставить значения выше 14 и посчитать, сколько их.

```python
temp = [12, 18, 9, 21, 15]
tepl = []
for t in temp:
    if t > 14:
        tepl.append(t)
print(tepl)        # [18, 21, 15]
print(len(tepl))   # 3
```

*Разбор:* это шаблон «фильтр» из 4.3 в чистом виде. Проходим циклом, условие `t > 14` отбирает подходящие: 18, 21, 15 проходят; 12 и 9 — нет (обрати внимание: 14 бы тоже не прошло, потому что строгое `>`). `append` собирает прошедшие в `tepl`, а количество даёт `len(tepl) = 3`. Что видим → почему: в результате `[18, 21, 15]` именно в том порядке, в каком они шли в исходном списке — цикл не сортирует, а сохраняет порядок.

**Пример 2 — среднее и понимание деления.** Что выведет `sum([10, 20, 30]) / len([10, 20, 30])` и почему?

*Разбор:* `sum([10, 20, 30])` равно `60`, `len([10, 20, 30])` равно `3`. Делим: `60 / 3 = 20.0`. Это среднее арифметическое. Обрати внимание на важную деталь: результат `20.0` **с точкой**, а не `20`. Деление `/` в Python всегда даёт дробное число (тип `float`), даже когда делится нацело. Если тебе нужно целое — есть отдельное деление `//` (`60 // 3` даст `20`), но для среднего почти всегда хотят именно дробное.

**Пример 3 — топ по полю в списке словарей.** Дан список товаров, нужно найти самый дорогой.

```python
tovary = [
    {"name": "Мышь", "cena": 1500},
    {"name": "Монитор", "cena": 45000},
    {"name": "Коврик", "cena": 800},
]
samyy_dorogoy = tovary[0]            # пока считаем первый самым дорогим
for t in tovary:
    if t["cena"] > samyy_dorogoy["cena"]:
        samyy_dorogoy = t            # нашли дороже — запомнили
print(samyy_dorogoy["name"])         # Монитор
```

*Разбор:* это классический поиск максимума, но не по числу, а по полю словаря. Приём такой: берём первую запись за «текущего лидера» (`tovary[0]`), затем циклом сравниваем каждую следующую по ключу `"cena"`; если очередная дороже лидера — она становится новым лидером. Проследи по шагам: лидер Мышь (1500) → Монитор 45000 больше, лидер Монитор → Коврик 800 меньше, лидер не меняется. В конце в `samyy_dorogoy` лежит запись `{"name": "Монитор", "cena": 45000}`, печатаем её имя — `Монитор`. Так же ищут студента с высшим баллом, заказ с максимальной суммой и так далее.

## 6. Частые ошибки и как их избежать

- **Ошибка:** обращаться к `spisok[3]` в списке из трёх элементов. **Почему:** индексы идут с нуля, у трёх элементов это `0, 1, 2`, а `3` уже нет — будет `IndexError`. **Как правильно:** помни, что последний индекс на единицу меньше длины; для последнего элемента бери `spisok[len(spisok) - 1]` или короткое `spisok[-1]`.
- **Ошибка:** делить `sum/len` без проверки на пустой список. **Почему:** на пустом списке `len` равно нулю — деление на ноль, `ZeroDivisionError`. **Как правильно:** сначала `if len(spisok) > 0:`, и только потом считать среднее.
- **Ошибка:** путать `=` и `==` в условии (`if o = 5`). **Почему:** `=` — присваивание, а в условии нужно сравнение `==`; Python выдаст `SyntaxError`. **Как правильно:** в `if` всегда двойное `==`.
- **Ошибка:** неверные отступы (`IndentationError` или код выполняется не там). **Почему:** в Python отступ определяет тело цикла и условия. **Как правильно:** ровно 4 пробела на каждый уровень вложенности; никогда не смешивай пробелы и табы.
- **Ошибка:** обращаться к словарю по индексу (`tovar[0]`) или к списку по ключу. **Почему:** это разные контейнеры с разным доступом; получишь `KeyError` или `TypeError`. **Как правильно:** список — по индексу `ceny[0]`, словарь — по ключу `tovar["name"]`.
- **Ошибка:** забыть обнулить накопитель (`itog`/счётчик) перед циклом. **Почему:** без `itog = 0` переменной ещё не существует — `NameError`. **Как правильно:** перед циклом всегда заводи накопитель: число `= 0`, список `= []`.
- **Ошибка:** взять `>` там, где нужно `>=` (или наоборот) — и тихо потерять или добавить граничное значение. **Почему:** `1800 > 1800` ложь, а `1800 >= 1800` истина — ошибка не падает, но результат неверный. **Как правильно:** на «дороже/старше/больше N» сразу уточняй: включать N или нет.

## 7. Памятка / практический алгоритм

Алгоритм обработки любого списка данных — держи под рукой:
1. **Выбери контейнер** — одно значение (переменная), набор (список), запись с полями (словарь), много записей (список словарей).
2. **Защити от пустоты** — `if len(spisok) > 0:` перед делением и перед доступом по индексу.
3. **Посчитай показатели** — `sum`, `len`, среднее (`sum/len`), `max`, `min`.
4. **Отфильтруй нужное** — шаблон «пустой список → `for` → `if` → `append`».
5. **Выведи и поясни** — `print` с подписями; проверь результат на крайних случаях (пусто, один элемент, все одинаковые).
6. **Соблюди приватность** — для учёбы и ИИ только обезличенные или тестовые данные (нормы РК).

## 8. Краткие итоги

- Контейнер выбирай по задаче: переменная — одно значение, список — набор однотипных значений, словарь — запись с подписанными полями, список словарей — много записей.
- К списку обращаются по индексу с нуля (`ceny[0]`), к словарю — по ключу (`tovar["name"]`); это разные способы доступа, не путай.
- Цикл `for` проходит по каждому элементу; отступ (4 пробела) задаёт тело цикла — это часть синтаксиса, а не оформление.
- Условие `if` отбирает подходящее; в нём всегда сравнение `==`, не путай с присваиванием `=`; следи за `>` и `>=`.
- Показатели считай готовыми функциями `sum`, `len`, `max`, `min`; среднее — это `sum/len` и всегда даёт дробное число.
- Фильтр собирают по шаблону: пустой список → `for` → `if` → `append`. Выучи его наизусть.
- Всегда проверяй пустые данные перед делением и доступом по индексу — иначе программа упадёт на реальных данных.
- С реальными данными людей работай только в обезличенном виде (нормы РК по персональным данным).

## 10. Глоссарий

- **append** — метод списка, добавляющий элемент в его конец (`spisok.append(x)`).
- **for (цикл)** — конструкция, повторяющая блок кода для каждого элемента набора.
- **if (условие)** — конструкция, выполняющая блок кода только при истинности проверки.
- **IndexError** — ошибка обращения к несуществующему индексу списка.
- **len** — функция, возвращающая количество элементов (длину) списка, строки, словаря.
- **max / min** — функции, возвращающие наибольший и наименьший элемент.
- **sum** — функция, возвращающая сумму всех чисел списка.
- **ZeroDivisionError** — ошибка деления на ноль.
- **Граничный (крайний) случай** — необычные входные данные (пустой список, ноль), на которых код часто ломается.
- **Индекс** — порядковый номер элемента списка; нумерация с нуля.
- **Ключ** — имя поля в словаре, по которому достают значение.
- **Контейнер данных** — способ хранения значений (переменная, список, словарь).
- **Накопитель** — переменная (число или список), в которой по ходу цикла накапливают результат.
- **Отступ** — пробелы в начале строки, задающие тело цикла или условия в Python (4 пробела на уровень).
- **Переменная** — именованная ячейка для одного значения; `=` — присваивание.
- **Присваивание (`=`)** — операция «положить значение справа в переменную слева».
- **Словарь (dict)** — контейнер пар «ключ — значение»; доступ по ключу.
- **Сравнение (`==`)** — проверка равенства двух значений в условии.
- **Среднее арифметическое** — сумма значений, делённая на их количество (`sum/len`).
- **Список (list)** — упорядоченный контейнер значений; доступ по индексу.
