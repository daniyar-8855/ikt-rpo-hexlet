# Структура дисциплины «Применение ИКТ и цифровых технологий» — РПО

Источник: рабочая учебная программа модуля ИКТ (ООМ 2), ТОО «Колледж Хекслет».

- Специальность: 06130100 «Программное обеспечение (по видам)».
- Квалификация: 4S06130103 «Разработчик программного обеспечения».
- Форма обучения: очная, на базе основного среднего образования.
- Общий объём: **96 часов аудиторных, 4 кредита** + **72 часа СРС**. Семестр 3. Итог — зачёт.
- Распределение по РО (по важности): РО 2.1 — 24 ч, РО 2.2 — 24 ч, РО 2.3 — 48 ч.
- Одна тема = одно занятие = одна папка урока (часы темы указаны в `reestr.md`).

## Файлы дисциплины

```
04_Курс_Moodle/
  structure.md            # этот файл
  reestr.md               # реестр занятий: часы, тип, статус
  NN_название-занятия/
    urok.md               # теория (источник истины)
    urok.html             # HTML-версия теории → Moodle «Страница»
    upr.md                # упражнения (закрепление)
    test.gift             # тест 7–10 вопросов → Moodle «Тест»
    zadanie.html          # условие задания + рубрика → Moodle «Задание»
    plan.md               # поурочный план (для преподавателя)
    assets/               # изображения (.svg инлайн)
```

> Сборка в Moodle — через MCP-сервер по инструкции методологического репозитория `Все для работы claude.ИИ` (политика: собирать только по явной просьбе).

## Стандарт заданий и тестов (повышенная сложность + анти-списывание)

- **Задания (`zadanie.html`):** для практических/проектных тем — **варианты по подгруппам** (4–6 кейсов, один на ~5 человек, раздача по номеру в журнале) + внутри каждого варианта **обязательная часть** (до 85 б.) и **продвинутая** (+15 б., устная защита). Задачи и рубрика единые. Лекционные темы — 1–2 варианта.
- **Тесты (`test.gift`):** банк **12–15 вопросов** на занятие; в Moodle тест берёт **случайные 10** из категории и перемешивает порядок (нативная защита от списывания). Только `multichoice`/`shortanswer`, вопросы — сценарные, на применение.
- **Оценивание — AI-устойчивое:** приоритет устной защите проектов/практик; письменные работы дополняются устной защитой.
- **Каждый урок (`urok.md`/`urok.html`) завершается блоком «Полезные ссылки»** — 3–5 проверенных источников (документация, нормативные акты РК, авторитетные платформы) по теме занятия.
- **Гриф одобрения (как в НВТП):** в конце `urok`, `zadanie`, `plan` — строка *«Материал разработан рабочей группой ТОО «Колледж Хекслет Казахстан» и одобрен к использованию в обучении решением Педагогического совета.»* В `plan` дополнительно — блок «Адаптация (при необходимости)».
