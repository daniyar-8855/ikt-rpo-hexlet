# Промпт-инжиниринг для кодовых задач

Один и тот же ИИ на расплывчатый запрос даёт бесполезный ответ, а на точный — почти готовое решение. Разница — в **промпте** (запросе). Промпт-инжиниринг — это навык формулировать запросы так, чтобы получать качественный результат. Для разработчика это прямой множитель продуктивности.

## Чему ты научишься

- строить структурированный промпт для кодовой задачи;
- применять приёмы: роль, контекст, примеры, формат ответа;
- итеративно улучшать промпт по результату.

## Анатомия хорошего промпта

| Часть | Зачем | Пример |
|---|---|---|
| Роль/контекст | задать рамку | «Ты опытный Python-разработчик» |
| Задача | что нужно | «напиши функцию валидации e-mail» |
| Вход/выход | конкретика | «на входе строка, на выходе True/False» |
| Ограничения | рамки | «только стандартная библиотека» |
| Формат ответа | как оформить | «код + краткое объяснение» |

## Приёмы промпт-инжиниринга

- **Конкретика вместо общего:** «функция суммы чётных» вместо «посчитай числа».
- **Примеры (few-shot):** дай 1–2 примера вход→выход.
- **Пошаговость:** «объясни по шагам, потом дай код».
- **Итерация:** не получилось — уточни запрос, добавь деталей/пример ошибки.

### Мини-кейс
Запрос «сделай парсер» дал бесполезный ответ. Уточнённый: «Напиши на Python функцию, которая из строки `имя=значение` возвращает словарь; пример: `a=1;b=2` → `{'a':'1','b':'2'}`; без библиотек» — дал рабочее решение. Следующий шаг: всегда добавлять пример вход→выход.

## Частые ошибки

**Ошибка:** слишком общий промпт («помоги с кодом»).
**Почему:** ИИ не знает деталей — ответ мимо.
**Как правильно:** язык, задача, вход/выход, ограничения, пример.

**Ошибка:** принять первый ответ и не итерировать.
**Почему:** часто нужна 1–2 уточняющих итерации.
**Как правильно:** уточнять промпт по результату (добавить пример, ошибку, ограничение).

## Итог: что теперь делать

- Структурируй промпт: роль, задача, вход/выход, ограничения, формат.
- Добавляй примеры вход→выход — это резко улучшает ответ.
- Итерируй: уточняй запрос, пока результат не станет рабочим.
- Помни: даже идеальный промпт не отменяет проверку кода.
