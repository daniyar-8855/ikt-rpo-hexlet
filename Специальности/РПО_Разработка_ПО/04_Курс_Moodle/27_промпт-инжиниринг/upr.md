# Упражнения. Занятие 27. Промпт-инжиниринг для кодовых задач

> Формат — составление и улучшение промптов. Критерии приёмки под каждым заданием.

## Упражнение 1. Разбери промпт на части
Возьми промпт «Ты Python-разработчик. Напиши функцию проверки e-mail: строка → True/False, только стандартная библиотека, дай код и пояснение». Найди роль, задачу, вход/выход, ограничение, формат.

**Критерий приёмки:** все 5 частей выделены верно.

## Упражнение 2. Улучши плохой промпт
Перепиши «помоги с кодом для парсинга» в структурированный промпт с примером вход→выход.

**Критерий приёмки:** есть язык, задача, вход/выход, ограничение и пример.

## Упражнение 3. Добавь пример (few-shot)
К задаче «привести строку к заголовочному регистру» добавь 2 примера вход→выход.

**Критерий приёмки:** 2 корректных примера (например `hello world` → `Hello World`).

## Упражнение 4. Итерация
Первый ответ ИИ не обработал пустую строку. Как уточнить промпт, чтобы это исправить?

**Критерий приёмки:** добавить требование/пример обработки пустого ввода и крайних случаев.

## Упражнение 5. Оцени два промпта
Сравни «сделай сортировку» и «Напиши на Python функцию сортировки списка чисел по убыванию, без сторонних библиотек; пример [3,1,2]→[3,2,1]». Какой лучше и почему?

**Критерий приёмки:** второй; конкретен, есть язык, направление сортировки, ограничение и пример.

## СРС (самостоятельная работа, 2 ч)

**Тема:** Библиотека промптов для кодовых задач

Собери 5–7 эффективных промптов для типовых кодовых задач с пояснением. Используй обезличенные/тестовые данные; если применял ИИ — укажи, для чего именно.

**Критерий приёмки:** промпты содержат роль, задачу, контекст и формат, есть пояснение чем хорош каждый. Работа сдаётся преподавателю (файл/отчёт) и засчитывается как СРС по теме.
