План учебного занятия

Организация: ТОО «Колледж Хекслет Казахстан»
Дисциплина: Применение информационно-коммуникационных и цифровых технологий
Тема занятия: Оформлять техническую документацию в текстовом редакторе
Подготовил педагог: ____________________
Дата: «___» __________ 20__ г.

1. Общие сведения

  • Курс, группа: ____________________ (квалификация 4S06130103 «Разработчик ПО»)
  • Тип занятия: комбинированное
  • Продолжительность: 2 часа (90 мин)
  • Результат обучения: РО 2.1 · Связь с ПМ: Разработка программных модулей

2. Цели, задачи

Цель: Научить структурно оформлять техническую документацию (README, инструкции) в текстовом редакторе и показать связь этого навыка с будущей профессией.

Задачи:

  • объяснить назначение технической документации и её отличие от «текста ради текста»;
  • показать средства структурирования: заголовки, списки, таблицы, код-блоки;
  • сформировать приём разметки заголовков стилями для автосборки оглавления;
  • отработать составление README по схеме (описание → установка → использование → ошибки).

3. Ожидаемые результаты

Обучающийся:

  • отличает техническую документацию от «текста ради текста»;
  • применяет стили заголовков, списки, таблицы и код-блоки;
  • объясняет, почему заголовки размечают стилями, а не «жирным» вручную;
  • составляет README по понятной схеме.

4. Необходимые ресурсы

  • Курс в Moodle: страница «Теория», тест и задание занятия 05.
  • Схемы: «Структура документа» и «Этапы подготовки документации» (папка assets).
  • Компьютер с текстовым редактором (Word/Google Docs) и редактором кода (Markdown), проектор, доступ к Moodle и интернету.
