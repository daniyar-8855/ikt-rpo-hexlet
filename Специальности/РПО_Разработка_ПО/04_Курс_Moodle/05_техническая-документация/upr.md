# Упражнения. Занятие 05. Техническая документация

> Формат — улучшение и разбор документов. Критерии приёмки под каждым заданием.

## Упражнение 1. Оцени README
Дан README: «Проект для учёбы. Запускается. Автор — Иван». Каких разделов не хватает, чтобы по нему смогли запустить проект?

**Критерий приёмки:** нет установки, использования (команды запуска), описания, раздела ошибок.

## Упражнение 2. Преврати «простыню» в инструкцию
Текст: «Скачайте архив распакуйте установите зависимости командой и запустите main потом откройте браузер». Перепиши как нумерованную инструкцию с код-блоками для команд.

**Критерий приёмки:** 4–5 пронумерованных шагов; команды вынесены в код-блок.

## Упражнение 3. Найди ошибку оформления
Студент сделал заголовки жирным текстом 16pt вручную. Почему это плохо и как исправить?

**Критерий приёмки:** не строится оглавление/навигация; решение — стили «Заголовок 1/2/3» (или `#` в Markdown).

## Упражнение 4. Подбери элемент оформления
Чем оформить каждый фрагмент: заголовок, список шагов, сравнение двух баз данных, команду установки?

**Критерий приёмки:** заголовок — стиль заголовка; шаги — нумерованный список; сравнение — таблица; команда — код-блок.

## Упражнение 5. Составь каркас README
Напиши только заголовки (структуру) README для любого учебного проекта по схеме из урока.

**Критерий приёмки:** есть Название, Описание, Установка, Использование, Возможные ошибки (порядок логичен).
