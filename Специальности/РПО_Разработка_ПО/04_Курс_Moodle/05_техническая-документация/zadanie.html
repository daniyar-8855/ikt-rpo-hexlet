Задание к занятию 05. Техническая документация проекта

Вариант — по номеру в журнале (свой тип проекта на ~5 человек). № 1–5 → В1 · 6–10 → В2 · 11–15 → В3 · 16–20 → В4 · 21–25 → В5 · 26–30 → В6.

Обязательная часть (до 85 баллов)

  1. Составь README для проекта своего варианта по схеме: Название → Описание → Установка → Использование → Возможные ошибки.
  2. Используй стили заголовков, нумерованные списки для шагов и код-блоки для команд.
  3. Добавь таблицу (например, параметры конфигурации или сравнение).
  4. Объём — 1–2 страницы; текст под действие, без «воды».
Продвинутая часть (+15 баллов): оформи README в Markdown (.md) с автоматическим оглавлением/якорями и покажи, как он отображается. Защищается устно.

Типы проектов по вариантам

  • В1. Веб-сайт-портфолио.
  • В2. Телеграм-бот.
  • В3. REST API-сервис.
  • В4. Десктоп-утилита.
  • В5. Скрипт анализа данных.
  • В6. Мобильное приложение.

Как сдать: загрузи документ/`.md`, укажи номер варианта. На занятии — короткая устная защита структуры README.

СРС (самостоятельная работа, 2 ч)

Тема СРС: Оформление техдокумента по стандарту.
Задание: Оформи технический документ (README или инструкцию) со стилями, заголовками и оглавлением. Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: есть заголовки-стили, оглавление и единое оформление.
