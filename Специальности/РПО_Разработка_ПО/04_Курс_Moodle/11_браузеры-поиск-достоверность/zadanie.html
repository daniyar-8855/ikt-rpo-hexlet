Задание к занятию 11. Поиск и проверка информации

Вариант — по номеру в журнале (свой запрос на ~5 человек). № 1–5 → В1 · 6–10 → В2 · 11–15 → В3 · 16–20 → В4 · 21–25 → В5 · 26–30 → В6.

Обязательная часть (до 85 баллов)

  1. Сформулируй точный поисковый запрос по теме своего варианта и применил минимум 2 оператора.
  2. Найди 3 источника и оцени каждый по признакам (кто, когда, зачем, проверяемость) — таблицей.
  3. Определи самый надёжный и обоснуй выбор.
  4. Сравни найденное с официальной документацией — совпадает ли.
Продвинутая часть (+15 баллов): найди в выдаче или у ИИ заведомо устаревший/неверный ответ по теме и покажи, как ты его опроверг через документацию. Защищается устно.

Темы поиска по вариантам

  • В1. Как открыть файл в Python с указанием кодировки.
  • В2. Как сделать адаптивную вёрстку в CSS.
  • В3. Как настроить виртуальное окружение в Python.
  • В4. Как отправить HTTP-запрос из JavaScript (fetch).
  • В5. Как прочитать CSV в pandas.
  • В6. Как создать ветку и слить её в git.

Как сдать: загрузи отчёт со скриншотами запроса/источников, укажи номер варианта. На занятии — короткая устная защита оценки источников.

СРС (самостоятельная работа, 2 ч)

Тема СРС: Подборка источников с оценкой достоверности.
Задание: Подбери 5 источников по теме и оцени их достоверность по критериям. Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: для каждого источника оценка по 4 критериям (автор, дата, издатель, проверяемость).
