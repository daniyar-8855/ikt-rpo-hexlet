**Тема занятия:** Автоматизация через API

## 1. Общие сведения

- **Курс, группа:** ____________________ (квалификация 4S06130103 «Разработчик ПО»)
- **Тип занятия:** комбинированное
- **Продолжительность:** 4 часа (2 пары, 180 мин)
- **Результат обучения:** РО 2.3 · **Связь с ПМ:** Разработка программных модулей

## 2. Цели, задачи

**Цель:** Сформировать понимание API как способа автоматизации обмена данными между программами; научить читать структуру запроса и ответа, различать методы GET и POST и применять это в учебных задачах.

**Задачи:**
- объяснить, что такое API, на аналогии «кафе и меню»;
- раскрыть структуру запроса (URL, метод, параметры) и формат ответа (JSON);
- научить различать методы GET и POST;
- показать применение API для автоматизации рутины и правила безопасного хранения ключа.

## 3. Ожидаемые результаты

Обучающийся:
- объясняет, что такое API и зачем он нужен;
- называет три части запроса и формат ответа;
- различает методы GET и POST и выбирает подходящий;
- понимает, как через API автоматизируют задачи, и как безопасно хранить ключ доступа.

## 4. Необходимые ресурсы

- Курс в Moodle: страница «Теория», тест и задание занятия 34.
- Схемы: «API как меню» и «GET и POST» (папка assets).
- Компьютер, проектор, доступ к Moodle и интернету.

## 5. Ход занятия

### Пара 1 — Что такое API и структура запроса (90 мин)

| Этап / Время | Действия педагога | Действия обучающегося |
|---|---|---|
| Организационный момент (5 мин) | Приветствие, проверка готовности и присутствующих | Занимают места, готовятся к занятию |
| Мотивация (10 мин) | Вводит практической ситуацией (ручное копирование курса валют), задаёт вопросы | Обсуждают, предлагают варианты |
| Целеполагание (5 мин) | Объявляет тему и цели занятия | Записывают тему, формулируют ожидания |
| Изучение нового материала (35 мин) | Раскрывает понятие API на аналогии «кафе/меню», демонстрирует схему «API как меню», разбирает запрос (URL, метод, параметры) и JSON-ответ | Слушают, работают со схемой, задают вопросы |
| Закрепление (15 мин) | Организует практику: разбор запроса и чтение JSON (упр. 1–3) | Выполняют задания, отвечают |
| Контроль (10 мин) | Проводит экспресс-опрос по структуре запроса | Отвечают, проверяют себя |
| Итог и рефлексия (5 мин) | Подводит итог первой пары, отвечает на вопросы | Подводят итог, уточняют |
| Задание (5 мин) | Поясняет задачу на пару 2 | Записывают, уточняют |

### Пара 2 — GET/POST, применение и контроль (90 мин)

| Этап / Время | Действия педагога | Действия обучающегося |
|---|---|---|
| Организационный момент (5 мин) | Проверка готовности, краткое повторение структуры запроса | Включаются в работу, отвечают |
