Задание к занятию 34. Автоматизация через API

Вариант — по номеру в журнале (свой публичный API на ~5 человек). № 1–5 → В1 · 6–10 → В2 · 11–15 → В3 · 16–20 → В4 · 21–25 → В5 · 26–30 → В6.

Обязательная часть (до 85 баллов)

  1. Выбери публичный API своего варианта, найди его документацию.
  2. Опиши запрос: URL, метод (GET/POST), параметры.
  3. Выполни запрос (в браузере, Postman или коде) и получи ответ.
  4. В ответе JSON найди 2–3 нужных поля по ключу и выпиши их значения.
  5. Опиши, какую рутину можно автоматизировать этим API.
Продвинутая часть (+15 баллов): напиши короткий скрипт (с ИИ-ассистентом), который делает запрос и выводит нужное поле, с проверкой успешности ответа. Объясни, как безопасно хранить ключ. Защити устно.

Публичные API по вариантам

  • В1. Открытые данные РК (data.egov.kz).
  • В2. Курсы валют (публичный API нацбанка/обменника).
  • В3. Погода (open-meteo или аналог).
  • В4. Случайные факты/цитаты (публичный API).
  • В5. Данные о странах (restcountries).
  • В6. Публичный API изображений/котиков/собак (для тренировки GET).

СРС (самостоятельная работа, 2 ч)

Тема СРС: Автоматизация задачи через публичный API.
Задание: Опиши или реализуй обращение к публичному API и извлеки нужные поля из JSON. Используй обезличенные данные; укажи, если применял ИИ.
Критерий приёмки: указаны URL, метод и параметры, извлечены поля, ключ хранится безопасно.
