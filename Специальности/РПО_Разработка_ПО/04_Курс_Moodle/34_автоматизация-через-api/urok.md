# Автоматизация через API

Программы не работают в одиночку — они обмениваются данными друг с другом. Приложение погоды берёт данные с сервера, бот в мессенджере отправляет сообщения, сайт магазина проверяет оплату в банке. Всё это работает через **API** — способ одной программы попросить что-то у другой. Понимание API превращает тебя из пользователя готовых программ в того, кто связывает их вместе и автоматизирует работу.

## Чему ты научишься

- понимать, что такое API и зачем он нужен;
- читать структуру запроса и ответа (URL, метод, JSON);
- видеть, как с помощью API автоматизируют рутину.

## Что такое API

**API** (Application Programming Interface) — это «окно», через которое одна программа обращается к другой по чётким правилам. Ты не знаешь, как сервис устроен внутри, но знаешь, **что попросить и что получишь в ответ**.

Аналогия: в кафе ты не идёшь на кухню — ты делаешь заказ официанту по меню. Меню — это и есть API: список того, что можно запросить, и в каком виде придёт ответ.

## Из чего состоит запрос

Обращение к API по сети (REST API) обычно содержит:

- **URL (адрес)** — куда обращаемся: `https://api.weather.com/today?city=Almaty`;
- **метод** — что делаем: `GET` (получить данные), `POST` (отправить/создать);
- **параметры** — уточнения запроса (город, дата);
- **ответ** — данные, чаще всего в формате **JSON**.

**JSON** — текстовый формат данных в виде пар «ключ: значение»:

```json
{
  "city": "Almaty",
  "temp": 18,
  "condition": "sunny"
}
```

Программа берёт из ответа нужное поле (`temp`) и использует его.

### Мини-кейс
Нужно каждое утро получать курс валют. Вручную — заходить на сайт. Через API — программа сама делает `GET`-запрос к сервису курсов, получает JSON, берёт нужное число и записывает в таблицу. Один раз настроил — дальше работает само.

## Где это применяют

- **получение данных** — погода, курсы, открытые данные (data.egov.kz);
- **отправка** — сообщения в мессенджер, запись в таблицу;
- **связка сервисов** — оплата, уведомления, синхронизация.

ИИ-ассистент помогает: можно описать задачу словами и попросить пример запроса к API.

## Частые ошибки

**Ошибка:** хранить ключ доступа (API-ключ) прямо в коде и выкладывать в открытый доступ.
**Почему:** ключ украдут и используют от твоего имени (расход, утечка).
**Как правильно:** хранить ключ отдельно (переменные окружения), не публиковать.

**Ошибка:** не проверять, что ответ успешный.
**Почему:** сервис может вернуть ошибку (нет данных, лимит) — программа сломается.
**Как правильно:** проверять код ответа и обрабатывать ошибку.

## Итог: что теперь делать

- Воспринимай API как «меню» запросов к чужой программе.
- Запрос — это URL + метод (GET/POST) + параметры; ответ обычно JSON.
- Бери из JSON нужное поле по ключу.
- Храни ключи доступа отдельно и проверяй успешность ответа.
