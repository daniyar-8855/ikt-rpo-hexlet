# Упражнения. Занятие 34. Автоматизация через API

> Формат — разбор запросов и ответов. Критерии приёмки под каждым заданием.

## Упражнение 1. Объясни API «своими словами»
Объясни, что такое API, на аналогии (например, кафе/меню).

**Критерий приёмки:** API — правила обращения одной программы к другой; знаешь, что попросить и что получишь, не зная устройства внутри.

## Упражнение 2. Разбери запрос
Дан запрос `GET https://api.weather.com/today?city=Almaty`. Назови: метод, адрес, параметр.

**Критерий приёмки:** метод GET; адрес api.weather.com/today; параметр city=Almaty.

## Упражнение 3. Прочитай JSON
Из ответа `{"city":"Almaty","temp":18}` возьми температуру.

**Критерий приёмки:** поле temp = 18 (доступ по ключу).

## Упражнение 4. GET или POST
Что использовать: (а) получить список товаров; (б) добавить новый заказ?

**Критерий приёмки:** (а) GET (получить); (б) POST (создать/отправить).

## Упражнение 5. Найди риск
Студент выложил код с API-ключом на GitHub. В чём опасность и что делать?

**Критерий приёмки:** ключ украдут/используют от его имени; убрать ключ из кода (переменные окружения), отозвать и заменить ключ.

## СРС (самостоятельная работа, 2 ч)

**Тема:** Автоматизация задачи через публичный API

Опиши или реализуй обращение к публичному API и извлеки нужные поля из JSON. Используй обезличенные/тестовые данные; если применял ИИ — укажи, для чего именно.

**Критерий приёмки:** указаны URL, метод и параметры, извлечены поля, ключ хранится безопасно. Работа сдаётся преподавателю (файл/отчёт) и засчитывается как СРС по теме.
